title: NIGHT ESCAPE 2：脱出口まで逃げ切れ！
---

このページでは、ハンターに捕まったときの再スタートと、起動装置で開く脱出口、ゲームクリアを追加します。

<Prerequisites>

- [NIGHT ESCAPE 1：警備中のハンターから逃げろ！](../night-escape-1/index.mdx)を完了している

</Prerequisites>

<Checkpoint title="このページで作るもの">

- 捕まると`CAUGHT!`が表示され、レベルが最初から始まる
- 起動装置を作動させると、脱出口が使えるようになる
- 脱出口に入ると`ESCAPED!`が表示され、ゲームが停止する

</Checkpoint>

<Section title="捕まったら最初からやり直す" goal="ハンターに捕まるとメッセージが表示され、レベルが最初から始まるようにします。">

<Action title="捕獲範囲を追加する">

`BP_Hunter`に`Sphere Collision`を追加し、名前を`CaptureCollision`に変更します。

</Action>

<Action title="捕獲範囲を設定する">

`CaptureCollision`をハンターの体を囲む大きさへ調整し、次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="Overlapイベントを追加する">

`CaptureCollision`の`On Component Begin Overlap`を追加します。

</Action>

<Action title="プレイヤーか確認する">

`Other Actor`ピンから`Cast To BP_ThirdPersonCharacter`を追加します。

```text
CaptureCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

</Action>

<Action title="捕獲判定を無効にする">

Cast成功側から`CaptureCollision`の`Set Collision Enabled`を実行し、`New Type`を`No Collision`にします。

捕獲処理の重複を防ぐため、プレイヤーを捕まえた時点で判定を無効にします。

</Action>

<Action title="メッセージを表示する">

`Print String`を追加し、`In String`を`CAUGHT!`、`Duration`を`2.0`、`Text Color`を赤にします。

</Action>

<Action title="再スタートまで待つ">

`Print String`のあとに`Delay`を接続し、`Duration`を`2.0`にします。

</Action>

<Action title="レベルを開き直す">

`Delay`のあとに`Open Level (by Object Reference)`を接続し、`Level`欄の一覧から`L_NightEscape`を選びます。

</Action>

完成した接続は次のようになります。

```text
Cast成功
    ↓
Set Collision Enabled
Target: CaptureCollision
New Type: No Collision
    ↓
Print String
In String: CAUGHT!
Duration: 2.0
Text Color: 赤
    ↓
Delay
Duration: 2.0
    ↓
Open Level (by Object Reference)
Level: L_NightEscape
```

<Action title="捕獲処理を保存する">

`BP_Hunter`をCompileしてSaveします。

</Action>

<Action title="捕まる距離を調整する">

Playし、見た目に合う距離で捕まるように`CaptureCollision`を調整します。

</Action>

<Verify title="捕獲と再スタートを確認する">

- 捕まると`CAUGHT!`が表示される
- レベルが最初から始まり、ハンターが再び巡回する

</Verify>

<Recovery title="近づいてもCAUGHT!が表示されないとき">

- `CaptureCollision`の`Generate Overlap Events`が有効か確認する
- `Other Actor`がCastの`Object`につながっているか確認する
- 体が触れても反応しない場合は、`CaptureCollision`を大きくする
- `BP_Hunter`をCompileしてSaveする

</Recovery>

<Recovery title="離れているのに捕まるとき">

- `CaptureCollision`を小さくして、もう一度Playする

</Recovery>

<Recovery title="CAUGHT!は表示されるが再スタートしないとき">

- `Print String`のあとに`Delay`がつながっているか確認する
- `Delay`のあとに`Open Level (by Object Reference)`がつながっているか確認する
- `Level`に`L_NightEscape`が設定されているか確認する
- 一覧に`L_NightEscape`が表示されない場合は、レベルをSaveしてから選び直す

</Recovery>

</details>

<details>
<summary>答え</summary>

</Checkpoint>

<Section title="2つの起動装置で脱出口を開く" goal="左右の起動装置を両方作動させたときだけ、脱出口が使えるようにします。">

### 演習1

<Exercise>

`BP_EscapeSwitch`を左右の通路に1つずつ配置し、2つとも作動させたときだけ脱出口が開くようにしてください。

次の状態になれば完成です。

- 開始時は脱出口を使えない
- 片方の起動装置だけでは脱出口が開かない
- 2つの起動装置を作動させると脱出口が開く
- 各起動装置は1回だけ反応する
- 脱出口が開いたあとに入るとゲームクリアになる

<Hint>

まず、レベル上の`BP_EscapeSwitch`を複製し、左右の通路へ1つずつ配置します。2つとも、Detailsの`EscapeGate`に同じ`BP_EscapeGate`を割り当ててください。

</Hint>

<Hint>

`BP_EscapeGate`に、作動した起動装置の数を保存するInteger型の変数を作ります。`ActivateEscapeGate`が呼ばれるたびに値を1増やし、値が2になった場合だけ、脱出口を有効にする処理へ進めます。

</Hint>

<Answer>

`BP_EscapeGate`にInteger型の変数`ActivatedSwitchCount`を作り、初期値を`0`にします。

`ActivateEscapeGate`の先頭で、現在の`ActivatedSwitchCount`に`1`を足し、その結果を`ActivatedSwitchCount`へ保存します。続けて`ActivatedSwitchCount >= 2`を`Branch`の`Condition`へ接続します。

```text
ActivateEscapeGate
    ↓
ActivatedSwitchCount + 1
    ↓
Set ActivatedSwitchCount
    ↓
ActivatedSwitchCount >= 2
    ↓
Branch

True
    ↓
Set Collision Enabled
Target: EscapeCollision
New Type: Query Only
    ↓
Set Light Color
Target: EscapeLight
    ↓
Print String
In String: EXIT OPEN!
```

元からあった脱出口を有効にする処理は、`Branch`の`True`側へ移します。`False`側には何も接続しません。これにより、1つ目の起動装置では数だけが増え、2つ目が作動したときに脱出口が開きます。

レベルでは`BP_EscapeSwitch`を左右の通路に1つずつ置き、両方の`EscapeGate`に同じ`BP_EscapeGate`を割り当てます。各`BP_EscapeSwitch`は作動直後に自身の`SwitchCollision`を無効にするため、同じ起動装置で数が繰り返し増えることはありません。

Playして、1つ目を作動させても脱出口がオレンジ色のままであること、2つ目を作動させると`EXIT OPEN!`が表示され、脱出口の色が変わることを確認します。その後、脱出口へ入ってゲームクリアになれば完成です。

</Answer>

</Exercise>

</Section>

<Section title="時間が余ったらゲームをアレンジする" goal="既習機能を使い、見た目や遊び方を発展させます。">

すべて行う必要はありません。興味のある演習を選びましょう。

### 演習-発展1

<Exercise>

ゲームの舞台を決め、その舞台に合う壁、通路、ハンター、脱出口、ライトへ作り替えてください。

基本の巡回、追跡、捕獲、脱出が動いたまま、何を表したステージなのか見た目から分かれば完成です。

<Hint>

まず、「場所」「追いかけてくるもの」「脱出口」の3つを決めます。3つが同じ場面に見える組み合わせにしてください。

</Hint>

<Hint>

決めた舞台に合わせて、最初に壁や床の色・マテリアル、巡回中の`HunterLight`、`EscapeLight`を変更します。そのあと、必要ならハンターや脱出口の見た目を変更します。

</Hint>

<Answer>

例えば、舞台を「停電した宇宙基地」にします。

```text
場所：停電した宇宙基地
ハンター：赤い警告灯を点灯する警備ロボット
脱出口：青く光る緊急転送ゲート
雰囲気：暗い通路で光を頼りに逃げる
```

壁や通路には金属製のアセットや暗いマテリアルを使い、巡回中のハンターは通常色、追跡時は赤色、脱出口は青色にします。ハンターと脱出口の光を見分けられるため、暗いステージでも進む方向が分かります。

見た目を変更したあとも、Navigation Meshが床全体につながり、警戒エリア、巡回、追跡、捕獲、脱出がすべて動くことを確認します。舞台の説明とステージの見た目が一致していれば完成です。

</Answer>

</Exercise>

### 演習-発展2

<Exercise>

これまでの授業で作った機能から1つ選び、逃走や脱出が面白くなるように追加してください。

例：

- 動く床
- スイッチ
- 物理オブジェクト
- Niagara
- 効果音
- UI
- チェックポイント
- シーケンサー

追加後も、巡回、追跡、捕獲、脱出がすべて動けば完成です。

<Hint>

以前のプロジェクトで動作しているBlueprintから、今回使いたいものを1つ選びます。まず`L_NightEscape`へ配置し、その機能だけが正しく動くか確認してください。

</Hint>

<Hint>

機能が単独で動いたら、ゲーム中のどこで使うかを決めます。動く床やスイッチは逃走経路、Niagaraやライトは脱出口、効果音は警戒開始や脱出、UIは操作方法や状態表示に使えます。配置後は、通路やハンターのNavigation Meshをふさいでいないか確認します。

</Hint>

<Answer>

一例として、スイッチを押すと近道が開く仕掛けを追加します。既習のスイッチBlueprintから、通路をふさいでいる扉や床のCustom Eventを呼び出し、プレイヤーが操作したときだけ近道を通れるようにします。

ハンターに追われながらスイッチへ向かう必要がある位置へ配置すると、単なる装飾ではなく逃走中の選択になります。近道が開く前にも脱出口へ到達できる通常ルートを残しておくと、仕掛けが動かない場合でもゲーム全体を確認できます。

Playして、スイッチを使わない場合と使った場合で逃げ方が変わること、追加した扉や床がNavigation Meshやハンターの移動を壊していないこと、捕獲と脱出が引き続き動くことを確認します。別の既習機能を選んだ場合も、ゲーム中の役割と動作確認が説明できれば完成です。

</Answer>

</Exercise>

ハンターごとに巡回地点と警戒エリアを用意し、Playしたときに役割の違いが分かれば完成です。

<Hint>

まず、レベル上の`BP_Hunter`、2つの巡回地点、`BP_AlertZone`をそれぞれ複製します。どちらのハンター用か分かる名前に変更すると、割り当てを間違えにくくなります。

</Hint>

<Hint>

2体目の`BP_Hunter`には2体目用の巡回地点を割り当て、2体目用の`BP_AlertZone`の`Hunter`にも2体目の`BP_Hunter`を割り当てます。1体目と2体目で、巡回地点と`Hunter`の組み合わせが混ざっていないか確認してください。

</Hint>

<Answer>

例えば、1体目は入口に近い場所をゆっくり巡回し、2体目は脱出口に近い場所を速く巡回させます。それぞれに別の`PatrolPointA`と`PatrolPointB`を割り当て、巡回経路が重なりすぎないようにします。

警戒エリアもハンターごとに用意します。各`BP_AlertZone`の`Hunter`には、その警戒エリアで追跡を始める`BP_Hunter`を割り当てます。同じハンターを両方へ割り当てると、もう1体が追跡を始めないため注意します。

Playして、警戒エリアに入る前は2体が別の場所を巡回し、それぞれの警戒エリアへ入ると対応するハンターが追跡を始めることを確認します。2体の開始位置、経路、速度の違いがプレイ中に分かれば完成です。

</Answer>

</Exercise>

### 演習-発展4

<Exercise>

捕まったときと脱出したときの`Print String`を、ゲームの結果がはっきり伝わる演出へ置き換えてください。

UI、アニメーション、音、カメラ、Niagaraなど、これまでに学んだ機能を使えます。少なくとも捕獲と脱出で違う演出になり、結果を見分けられれば完成です。

<Hint>

まず、捕獲用と脱出用のどちらか一方だけを作ります。以前作ったWidgetを参考に、画面中央へ大きな文字を表示し、`Create Widget`と`Add to Viewport`で表示できるか確認してください。

</Hint>

<Hint>

捕獲処理では、Widgetを表示したあとに既存の`Delay`と`Open Level`へつなぎます。脱出処理では、Widgetを表示したあとに`Set Game Paused`へつなぎます。演出が動くことを確認してから、対応する`Print String`を削除してください。

</Hint>

<Answer>

一例として、捕獲用の`WBP_Caught`と脱出用の`WBP_Escaped`を作ります。どちらも画面中央に大きな文字を置き、捕獲は赤色、脱出は緑または水色にします。背景を半透明にしたり、Widget Animationで文字を拡大・フェードインさせたりすると、結果がさらに分かりやすくなります。

捕獲処理では、`Print String`の代わりに`Create Widget`で`WBP_Caught`を作り、`Add to Viewport`へ接続します。そのあとに、既存の`Delay`と`Open Level (by Object Reference)`を接続します。

```text
捕獲判定を無効にする
    ↓
Create Widget: WBP_Caught
    ↓
Add to Viewport
    ↓
Delay: 2.0
    ↓
Open Level: L_NightEscape
```

脱出処理では、`Print String`の代わりに`WBP_Escaped`を表示し、そのあとに`Set Game Paused`を実行します。

```text
脱出判定を無効にする
    ↓
Create Widget: WBP_Escaped
    ↓
Add to Viewport
    ↓
Set Game Paused
```

必要に応じて、Widgetを表示する前に捕獲音や脱出音を再生します。捕獲では画面を赤く点滅させ、脱出ではNiagaraや明るい効果音を使うなど、結果に合う演出にしてください。

Playして、捕まったときに捕獲用の演出が表示されたあと再スタートすること、脱出口へ入ったときに脱出用の演出が表示されてゲームが停止することを確認します。両方の結果を一目で見分けられ、元のゲーム進行が壊れていなければ完成です。

</Answer>

</Exercise>
