---
title: NIGHT ESCAPE 2：脱出口まで逃げ切れ！
---

このページでは、ハンターに捕まったときの再スタートと、脱出口に入ったときのゲームクリアを追加します。

<Prerequisites>

- [NIGHT ESCAPE 1：警備中のハンターから逃げろ！](../night-escape-1/index.mdx)を完了している
- `L_NightEscape`でハンターが巡回する
- プレイヤーが`BP_AlertZone`に入るとハンターが追いかける
- ハンターが壁や曲がり角を回り込んで移動する

</Prerequisites>

<Checkpoint title="このページで作るもの">

- 捕まると`CAUGHT!`が表示され、レベルが最初から始まる
- 脱出口に入ると`ESCAPED!`が表示され、ゲームが停止する
- 捕獲と脱出の両方をPlayで確認する

</Checkpoint>

---

<Section title="捕まったら最初からやり直す" goal="ハンターに捕まるとメッセージが表示され、レベルが最初から始まるようにします。">

<Action title="捕獲範囲を追加する">

`BP_Hunter`を開き、Componentsパネルで`Sphere Collision`を追加します。

名前を`CaptureCollision`に変更し、ハンターの体を囲むように大きさと位置を調整します。捕まる距離は、処理を作ったあとにPlayしながら調整します。

`CaptureCollision`を次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="プレイヤーが入ったか調べる">

Componentsパネルで`CaptureCollision`を選び、DetailsのEventsから`On Component Begin Overlap`を追加します。

`Cast To BP_ThirdPersonCharacter`を追加し、`Other Actor`をCastの`Object`に接続します。

```text
CaptureCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

捕獲範囲に入ったActorがプレイヤーかどうかを、Castで確認します。

Cast成功側から、`CaptureCollision`の`Set Collision Enabled`を実行します。`New Type`は`No Collision`にします。

捕獲処理が繰り返されないように、プレイヤーを捕まえた時点で判定を無効にします。

</Action>

<Concept title="画面に文字を表示する">

`Print String`は、画面の左上に文字を表示するノードです。`In String`に表示する文字、`Duration`に表示する秒数を指定します。

</Concept>

<Concept title="少し待ってからレベルを開き直す">

`Delay`は、指定した秒数だけ待ってから次の処理へ進むノードです。`Open Level (by Object Reference)`は、指定したレベルを開くノードです。

今回は、`L_NightEscape`を開き直して、レベル内のActorをゲーム開始時の状態へ戻します。

</Concept>

<Action title="メッセージを表示して再スタートする">

`Set Collision Enabled`のあとに`Print String`を追加し、次のように設定します。

| 項目        | 値        |
| ----------- | --------- |
| `In String` | `CAUGHT!` |
| `Duration`  | `0.5`     |

続けて、`Delay`と`Open Level (by Object Reference)`を追加します。

| ノード                             | 設定                   |
| ---------------------------------- | ---------------------- |
| `Delay`                            | `Duration: 0.5`        |
| `Open Level (by Object Reference)` | `Level: L_NightEscape` |

`Open Level (by Object Reference)`の`Level`欄をクリックし、一覧から`L_NightEscape`を選びます。

`Print String`の表示時間と`Delay`の待ち時間をどちらも`0.5`秒にすると、`CAUGHT!`の表示が終わる頃にレベルを開き直せます。

実行ピンを、次の順番で接続します。

```text
Cast成功
    ↓
Set Collision Enabled
Target: CaptureCollision
New Type: No Collision
    ↓
Print String
In String: CAUGHT!
Duration: 0.5
    ↓
Delay
Duration: 0.5
    ↓
Open Level (by Object Reference)
Level: L_NightEscape
```

`BP_Hunter`をCompileしてSaveします。

</Action>

<Action title="捕まる距離を調整する">

Playして警戒エリアに入り、ハンターへ近づきます。

- 離れているのに捕まる場合は、`CaptureCollision`を小さくする
- 体が触れているように見えても捕まらない場合は、`CaptureCollision`を大きくする

見た目として自然な距離で`CAUGHT!`が表示されるまで調整してください。

</Action>

<Verify title="捕獲と再スタートを確認する">

1. 警戒エリアに入る
2. ハンターに近づく
3. `CAUGHT!`が表示される
4. `L_NightEscape`が最初から始まる
5. ハンターが再び巡回する

</Verify>

<Recovery title="近づいてもCAUGHT!が表示されないとき">

- `CaptureCollision`の`Generate Overlap Events`が有効か確認する
- `Other Actor`がCastの`Object`につながっているか確認する
- `CaptureCollision`を、プレイヤーが入れる大きさにする
- `BP_Hunter`をCompileしてSaveする

</Recovery>

<Recovery title="CAUGHT!は表示されるが再スタートしないとき">

- `Print String`のあとに`Delay`がつながっているか確認する
- `Delay`のあとに`Open Level (by Object Reference)`がつながっているか確認する
- `Level`に`L_NightEscape`が設定されているか確認する
- 一覧に`L_NightEscape`が表示されない場合は、`L_NightEscape`をSaveしてから`Level`を選び直す

</Recovery>

<Checkpoint title="理解度確認1">

`Other Actor`を`Cast To BP_ThirdPersonCharacter`に接続するのはなぜですか。

<details>
<summary>ヒント</summary>

捕獲範囲には、プレイヤー以外のActorが入る場合もあります。

</details>

<details>
<summary>答え</summary>

捕獲範囲に入ったActorがプレイヤーであることを確認し、プレイヤーが入った場合だけ捕獲処理を実行するためです。

</details>

</Checkpoint>

<Checkpoint title="理解度確認2">

`Print String`のあとに`Delay`を入れるのはなぜですか。

<details>
<summary>ヒント</summary>

`Open Level`を実行すると、画面がすぐに開始時の状態へ変わります。

</details>

<details>
<summary>答え</summary>

レベルを開き直す前に、`CAUGHT!`を画面に表示する時間を作るためです。

</details>

</Checkpoint>

</Section>

<Section title="脱出口を作る" goal="プレイヤーが中に入れる、光る脱出口を作ります。">

<Action title="脱出口のBlueprintを作る">

`Content/NightEscape/Blueprints`に、`Actor`を親にしたBlueprintを作り、名前を`BP_EscapeGate`にします。

次のコンポーネントを追加します。

| コンポーネント  | 名前              |
| --------------- | ----------------- |
| `Cube`          | `GateMesh`        |
| `Box Collision` | `EscapeCollision` |
| `Point Light`   | `EscapeLight`     |

</Action>

<Action title="脱出口の見た目と判定範囲を作る">

`GateMesh`を縦長に調整し、入口や転送装置のような形にします。

`GateMesh`の`Collision Presets`を`NoCollision`にします。見た目には`GateMesh`を使い、プレイヤーが入ったかどうかは`EscapeCollision`で判定します。

`EscapeCollision`を、プレイヤーが通れる大きさに広げます。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

`EscapeLight`は、脱出口の場所が分かる色と明るさにします。ハンターのライトとは違う色にすると見分けやすくなります。

</Action>

<Action title="脱出口をレベルに配置する">

`BP_EscapeGate`を、プレイヤーの開始位置から通路を進んだ先に配置します。

ハンターの巡回場所や警戒エリアから少し離し、追いかけられてから逃げる時間を作ります。

</Action>

<Verify title="脱出口に入れるか確認する">

Playして確認します。

- `EscapeLight`の光で脱出口の場所が分かる
- `GateMesh`に移動を遮られない
- プレイヤーが`EscapeCollision`の中に入れる
- ハンターのライトと脱出口のライトを見分けられる

</Verify>

</Section>

<Section title="脱出口に入ったらゲームクリアにする" goal="脱出口に入るとメッセージが表示され、ゲームが停止するようにします。">

<Action title="プレイヤーが入ったか調べる">

`BP_EscapeGate`を開き、`EscapeCollision`の`On Component Begin Overlap`を追加します。

`Cast To BP_ThirdPersonCharacter`を追加し、`Other Actor`をCastの`Object`に接続します。

```text
EscapeCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

</Action>

<Action title="脱出処理が繰り返されないようにする">

Cast成功側から、`EscapeCollision`の`Set Collision Enabled`を実行します。

`New Type`を`No Collision`にします。

</Action>

<Concept title="ゲームの動きを一時停止する">

`Set Game Paused`は、Playを終了せず、現在のゲームの動きを一時停止するノードです。停止後にPlayを終了するには、エディタのStopボタンを使います。

</Concept>

<Action title="メッセージを表示してゲームを停止する">

`Print String`を追加し、次のように設定します。

| 項目        | 値         |
| ----------- | ---------- |
| `In String` | `ESCAPED!` |
| `Duration`  | `10`       |

ゲームを停止した直後も結果を確認しやすいように、`ESCAPED!`の表示時間は`10`秒にします。

そのあとに`Set Game Paused`を追加し、`Paused`を有効にします。

```text
Cast成功
    ↓
Set Collision Enabled
Target: EscapeCollision
New Type: No Collision
    ↓
Print String
In String: ESCAPED!
Duration: 10
    ↓
Set Game Paused
Paused: true
```

`BP_EscapeGate`をCompileしてSaveします。

</Action>

<Verify title="ゲームクリアを確認する">

Playして、ハンターから逃げながら脱出口に入ります。

- `ESCAPED!`が表示される
- プレイヤーの動きが止まる
- ハンターの動きが止まる
- エディタのStopボタンでPlayを終了できる

</Verify>

<Recovery title="脱出口に入ってもESCAPED!が表示されないとき">

- `GateMesh`ではなく、`EscapeCollision`の中に入っているか確認する
- `EscapeCollision`の`Generate Overlap Events`が有効か確認する
- `Other Actor`がCastの`Object`につながっているか確認する
- Cast成功側から`Print String`まで実行ピンがつながっているか確認する
- `BP_EscapeGate`をCompileしてSaveする

</Recovery>

<Recovery title="ESCAPED!は表示されるがゲームが停止しないとき">

- `Print String`のあとに`Set Game Paused`がつながっているか確認する
- `Set Game Paused`の`Paused`が有効になっているか確認する
- Cast成功側から`Set Game Paused`まで実行ピンが途切れていないか確認する
- `BP_EscapeGate`をCompileしてSaveする

</Recovery>

<Checkpoint title="理解度確認3">

脱出処理の最初に`EscapeCollision`を無効にするのはなぜですか。

<details>
<summary>ヒント</summary>

Overlapイベントが続けて発生した場合を考えます。

</details>

<details>
<summary>答え</summary>

`ESCAPED!`の表示やゲーム停止の処理が、重複して実行されるのを防ぐためです。

</details>

</Checkpoint>

</Section>

<Section title="捕獲と脱出を確認する" goal="捕まる場合と脱出する場合を分けてテストします。">

<Verify title="捕まる場合を確認する">

1. 警戒エリアに入る
2. ハンターに追いかけさせる
3. ハンターに捕まる
4. `CAUGHT!`が表示される
5. レベルが最初から始まり、ハンターが再び巡回する

</Verify>

<Verify title="脱出する場合を確認する">

1. 警戒エリアに入る
2. ハンターから逃げる
3. 脱出口に入る
4. `ESCAPED!`が表示される
5. プレイヤーとハンターが停止する

</Verify>

<Action title="レベルとBlueprintを保存する">

`BP_Hunter`、`BP_AlertZone`、`BP_EscapeGate`をCompileしてSaveします。

最後に`L_NightEscape`をSaveします。

</Action>

<Checkpoint title="完成チェック">

- ハンターが巡回し、警戒エリアに入ると追跡が始まる
- 捕まると`CAUGHT!`が表示され、最初から再開する
- 脱出口に入ると`ESCAPED!`が表示され、ゲームが停止する
- 捕獲と脱出の両方をPlayで確認した
- レベルとBlueprintを保存した

</Checkpoint>

</Section>

<Section title="時間が余ったらゲームをアレンジする" goal="基本のゲームを、自分のアイデアに合わせて作り替えます。">

ここからは発展演習です。基本のゲームが完成し、時間に余裕がある場合に取り組んでください。

すべて行う必要はありません。興味のある演習を選びましょう。

### 演習-発展1

<Exercise>

ゲームの舞台を決めます。

自分の`NIGHT ESCAPE`について、次の4項目を考えてください。

```text
場所：
ハンター：
脱出口：
目指す雰囲気：
```

例えば、次のようなゲームにできます。

- 停電した研究施設から、実験体を避けて脱出する
- 夜の迷路で、正体不明の怪物から逃げる
- 廃倉庫で、警備ロボットに追われながら出口を探す
- 宇宙基地で、故障した作業ロボットから逃げる

標準アセット、Cube、ライトを中心に表現できる内容にまとめてください。

<Hint>

「どこで、何から、どこへ逃げるか」を1つの場面として考えると、4項目を決めやすくなります。

</Hint>

<Answer>

次は一例です。

```text
場所：停電した宇宙基地
ハンター：赤い警告灯を点灯する警備ロボット
脱出口：青く光る緊急転送ゲート
目指す雰囲気：暗い通路で光を頼りに逃げる
```

4項目から同じ場面を想像でき、今回使ったBlueprintと標準アセットで表現できる内容なら完成です。

</Answer>

</Exercise>

### 演習-発展2

<Exercise>

決めた舞台に合わせて、`L_NightEscape`のステージを作り替えてください。

次の条件を満たします。

- スタートから脱出口まで進める
- 警戒エリアに入ると、ハンターがプレイヤーまで移動できる
- どちらへ進むか考える場所が1か所ある
- 舞台に合う見た目へ1か所以上変更している

<Hint>

最初に、スタート、警戒エリア、ハンターの巡回場所、脱出口を決めます。そのあとで、分かれ道や障害物を追加します。

</Hint>

<Answer>

例えば、次のような配置にすると、近道と遠回りを選べるステージになります。

```text
プレイヤー開始位置
        │
    警戒エリア
        │
      分かれ道
    ├────────┐
短くて危険な道    長くて安全な道
    └────────┤
             脱出口
```

見た目を作り込む前に、プレイヤーとハンターの両方が必要な場所まで移動できることを確認してください。

</Answer>

</Exercise>

### 演習-発展3

<Exercise>

自分のステージを3回Playし、難易度を調整してください。

1. 脱出口まで逃げる
2. 1回目とは別のルートを試す
3. ハンターに近づいて捕まる

気になった点を1つ選び、次の中から関連する設定だけを変更します。

- ハンターの速さ
- `CaptureCollision`の大きさ
- 巡回地点
- 警戒エリア
- 脱出口
- 壁や通路

変更後にもう一度Playし、狙った変化が出たか確認してください。

<Hint>

一度に複数の項目を変えると、どの変更が影響したのか分かりにくくなります。まずは1項目だけ変更します。

</Hint>

<Answer>

例えば、ハンターがすぐに追いつく場合は、ハンターの速さを下げるか、警戒エリアと脱出口の間を広げます。

反対に、ほとんど捕まらない場合は、ハンターの速さを上げるか、巡回場所を通路へ近づけます。

変更前と比べて、狙った方向に難易度が変われば完成です。

</Answer>

</Exercise>

### 演習-発展4

<Exercise>

これまでの授業で作った機能から、自分で追加できそうなものを1つ選びます。

例：

- 動く床を逃走経路に置く
- スイッチで新しい道を開く
- 物理オブジェクトを障害物にする
- Niagaraで脱出口を目立たせる
- 効果音で警戒や脱出を表現する
- UIで操作方法を表示する
- チェックポイントを追加する
- シーケンサーで開始時の演出を作る

追加後も、巡回、追跡、捕獲、脱出がすべて動けば完成です。

<Hint>

追加する機能が、「逃げる」「見つかる」「脱出する」のどこを面白くするのかを先に決めます。

</Hint>

<Answer>

例えば、スイッチで近道を開けるようにすると、ハンターから逃げながらスイッチを使う場面を作れます。

追加した機能が基本のゲーム進行を壊さず、遊び方や舞台の雰囲気を分かりやすくできれば完成です。

</Answer>

</Exercise>

### 演習-発展5

<Exercise>

`BP_Hunter`をもう1体配置し、2体に違う役割を持たせてください。

例えば、次のように役割を分けられます。

- 入口近くと脱出口近くを守る
- 近くにいる遅いハンターと、遠くにいる速いハンターにする
- 左右のルートを1体ずつ守る

ハンターごとに2つの巡回地点と`BP_AlertZone`を用意し、それぞれ正しいハンターを割り当てます。

Playして、2体の役割の違いが分かれば完成です。

<Hint>

開始位置、巡回経路、速さ、警戒エリアのどれかを変えると、2体の違いが伝わりやすくなります。

</Hint>

<Answer>

例えば、1体目を入口近くで遅く巡回させ、2体目を脱出口の近くで速く巡回させます。

それぞれの`BP_AlertZone`の`Hunter`に対応するハンターを割り当てると、警戒エリアごとに追跡を始めるハンターを分けられます。

</Answer>

</Exercise>

</Section>
