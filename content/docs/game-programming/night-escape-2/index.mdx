---
title: NIGHT ESCAPE 2：脱出口まで逃げ切れ！
---

このページでは、ハンターに捕まったときの再スタートと、脱出口に入ったときのゲームクリアを追加します。

<Prerequisites>

- [NIGHT ESCAPE 1：警備中のハンターから逃げろ！](../night-escape-1/index.mdx)を完了している
- `L_NightEscape`でハンターが巡回する
- プレイヤーが`BP_AlertZone`に入ると、ハンターが追いかける
- ハンターが壁や曲がり角を回り込んで移動する

</Prerequisites>

<Checkpoint title="このページで作るもの">

- ハンターに捕まると`CAUGHT!`が表示され、最初から再開する
- 脱出口に入ると`ESCAPED!`が表示され、ゲームが停止する
- 捕獲と脱出の両方をPlayで確認する

</Checkpoint>

---

<Section title="捕まったら最初からやり直す" goal="ハンターに捕まるとメッセージを表示し、レベルを最初から始めます。">

<Action title="捕獲範囲を追加する">

`BP_Hunter`を開き、Componentsパネルで`Sphere Collision`を追加します。

名前を`CaptureCollision`に変更し、ハンターの体を囲むように大きさと位置を調整します。捕まる距離は、処理を作ったあとにPlayしながら調整します。

`CaptureCollision`を次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="プレイヤーが入ったか調べる">

Componentsパネルで`CaptureCollision`を選び、DetailsのEventsから`On Component Begin Overlap`を追加します。

`Cast To BP_ThirdPersonCharacter`を追加し、`Other Actor`をCastの`Object`に接続します。

```text
CaptureCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

これで、捕獲範囲に入ったActorがプレイヤーかどうかを確認できます。

Cast成功側から、`CaptureCollision`の`Set Collision Enabled`を実行します。`New Type`は`No Collision`にします。

```text
Cast成功
    ↓
Set Collision Enabled
Target: CaptureCollision
New Type: No Collision
```

これで、捕まったあとの処理が繰り返されなくなります。

</Action>

<Action title="メッセージを表示して再スタートする">

`Set Collision Enabled`のあとに`Print String`を追加し、次のように設定します。

| 項目       | 値        |
| ---------- | --------- |
| `In String` | `CAUGHT!` |
| `Duration`  | `0.5`     |

続けて、`Delay`と`Open Level (by Object Reference)`を追加します。

`Delay`の`Duration`を`0.5`にし、`Open Level (by Object Reference)`の`Level`に`L_NightEscape`を設定します。

```text
Set Collision Enabled
    ↓
Print String
In String: CAUGHT!
Duration: 0.5
    ↓
Delay
Duration: 0.5
    ↓
Open Level (by Object Reference)
Level: L_NightEscape
```

`Delay`を入れることで、レベルを開き直す前に`CAUGHT!`を表示できます。

CompileしてSaveします。

</Action>

<Verify title="捕獲と再スタートを確認する">

Playして、次を確認します。

1. 警戒エリアに入る
2. ハンターに近づく
3. `CAUGHT!`が表示される
4. `L_NightEscape`が最初から始まる
5. ハンターが再び巡回する

見た目として自然な距離で捕まるように、`CaptureCollision`の大きさを調整してください。

</Verify>

<Recovery title="近づいてもCAUGHT!が表示されない">

- `CaptureCollision`のOverlapが有効か確認する
- `Other Actor`をCastの`Object`に接続しているか確認する
- Cast成功側から`Print String`まで実行ピンがつながっているか確認する
- プレイヤーが触れられる大きさに`CaptureCollision`を広げる

</Recovery>

<Recovery title="CAUGHT!は表示されるが再スタートしない">

- `Print String`のあとに`Delay`を接続しているか確認する
- `Delay`のあとに`Open Level (by Object Reference)`を接続しているか確認する
- `Level`に`L_NightEscape`を設定しているか確認する
- `L_NightEscape`をSaveしているか確認する

</Recovery>

<Recovery title="離れているのに捕まる">

`CaptureCollision`を小さくし、もう一度Playします。見た目と捕まる距離が合うまで、少しずつ調整してください。

</Recovery>

<Checkpoint title="理解度確認1">

`Other Actor`を`Cast To BP_ThirdPersonCharacter`に接続するのはなぜですか。

<details>
<summary>ヒント</summary>

捕獲範囲には、プレイヤー以外のActorが入ることもあります。

</details>

<details>
<summary>答え</summary>

捕獲範囲に入ったActorがプレイヤーであることを確認し、プレイヤーが入ったときだけ捕獲処理を実行するためです。

</details>

</Checkpoint>

<Checkpoint title="理解度確認2">

捕まったあとに`Delay`を入れるのはなぜですか。

<details>
<summary>ヒント</summary>

`Delay`の前で`CAUGHT!`を表示し、そのあとでレベルを開き直します。

</details>

<details>
<summary>答え</summary>

レベルを開き直す前に、`CAUGHT!`を0.5秒間表示するためです。

</details>

</Checkpoint>

</Section>

<Section title="脱出口を作る" goal="プレイヤーが入れる脱出口を作り、ステージの奥に配置します。">

<Action title="BP_EscapeGateを作る">

`Content/NightEscape/Blueprints`に、`Actor`を親にしたBlueprintを作り、名前を`BP_EscapeGate`にします。

</Action>

<Action title="脱出口の見た目と判定範囲を作る">

`BP_EscapeGate`に、次のコンポーネントを追加します。

| コンポーネント | 名前              |
| -------------- | ----------------- |
| `Cube`         | `GateMesh`        |
| `Box Collision` | `EscapeCollision` |
| `Point Light`  | `EscapeLight`     |

`GateMesh`を縦長にし、入口や転送装置のように見える形を作ります。

`GateMesh`の`Collision Presets`を`NoCollision`にします。見た目には`GateMesh`を使い、プレイヤーが入ったかどうかは`EscapeCollision`で判定します。

`EscapeCollision`をプレイヤーが入れる大きさに広げ、次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

`EscapeLight`は、ハンターのライトと見分けられる色と明るさにします。

</Action>

<Action title="レベルに配置する">

`BP_EscapeGate`を、プレイヤーの開始位置から通路を進んだ先に配置します。

ハンターの巡回場所や警戒エリアから少し離し、追いかけられてから脱出口まで逃げる時間を作ります。

</Action>

<Verify title="脱出口を確認する">

Playして、次を確認します。

- `EscapeLight`の光で脱出口の位置が分かる
- `GateMesh`に触れても移動を遮られない
- プレイヤーが`EscapeCollision`の中に入れる
- ハンターのライトと脱出口のライトを見分けられる

</Verify>

</Section>

<Section title="脱出口に入ったらゲームクリアにする" goal="脱出口に入るとメッセージを表示し、ゲームを停止します。">

<Action title="プレイヤーが入ったか調べる">

`BP_EscapeGate`を開き、`EscapeCollision`の`On Component Begin Overlap`を追加します。

`Cast To BP_ThirdPersonCharacter`を追加し、`Other Actor`をCastの`Object`に接続します。

```text
EscapeCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

</Action>

<Action title="脱出処理が繰り返されないようにする">

Cast成功側から、`EscapeCollision`の`Set Collision Enabled`を実行します。

`New Type`は`No Collision`にします。

</Action>

<Action title="ESCAPED!を表示してゲームを停止する">

`Set Collision Enabled`のあとに`Print String`を追加します。

| 項目       | 値         |
| ---------- | ---------- |
| `In String` | `ESCAPED!` |
| `Duration`  | `10`       |

続けて`Set Game Paused`を追加し、`Paused`を有効にします。

```text
Cast成功
    ↓
Set Collision Enabled
Target: EscapeCollision
New Type: No Collision
    ↓
Print String
In String: ESCAPED!
    ↓
Set Game Paused
Paused: true
```

CompileしてSaveします。

</Action>

<Verify title="ゲームクリアを確認する">

Playして、ハンターから逃げながら脱出口に入ります。

- `ESCAPED!`が表示される
- プレイヤーが停止する
- ハンターが停止する
- エディタのStopボタンでPlayを終了できる

</Verify>

<Recovery title="脱出口に入ってもESCAPED!が表示されない">

- `GateMesh`が`NoCollision`になっているか確認する
- `EscapeCollision`のOverlapが有効か確認する
- `Other Actor`をCastの`Object`に接続しているか確認する
- Cast成功側から`Print String`まで実行ピンがつながっているか確認する

</Recovery>

<Recovery title="ESCAPED!は表示されるがゲームが止まらない">

- `Print String`のあとに`Set Game Paused`を接続しているか確認する
- `Set Game Paused`の`Paused`が有効か確認する

</Recovery>

<Checkpoint title="理解度確認3">

脱出処理の最初に`EscapeCollision`を無効にするのはなぜですか。

<details>
<summary>ヒント</summary>

Overlapイベントは、判定範囲に入るたびに実行されます。

</details>

<details>
<summary>答え</summary>

`ESCAPED!`の表示やゲーム停止の処理が、繰り返し実行されないようにするためです。

</details>

</Checkpoint>

</Section>

<Section title="捕獲と脱出を確認する" goal="捕まる場合と脱出する場合を分けてテストします。">

<Action title="捕まる場合を確認する">

Playして、次の順番で確認します。

1. ハンターが2つの地点を巡回する
2. 警戒エリアに入ると追跡が始まる
3. ハンターに捕まると`CAUGHT!`が表示される
4. レベルが最初から始まる
5. ハンターが再び巡回する

</Action>

<Action title="脱出する場合を確認する">

もう一度Playして、次の順番で確認します。

1. 警戒エリアに入る
2. ハンターから逃げる
3. 脱出口に入る
4. `ESCAPED!`が表示される
5. プレイヤーとハンターが停止する

</Action>

<Action title="保存する">

`BP_Hunter`、`BP_AlertZone`、`BP_EscapeGate`をCompileしてSaveします。

`L_NightEscape`もSaveします。

</Action>

<Checkpoint title="完成チェック">

- ハンターが巡回し、警戒エリアに入ると追跡が始まる
- 捕まると`CAUGHT!`が表示され、最初から再開する
- 脱出口に入ると`ESCAPED!`が表示され、ゲームが停止する
- 捕獲と脱出の両方を確認した
- レベルとBlueprintを保存した

</Checkpoint>

</Section>

<Section title="時間が余ったらゲームをアレンジする" goal="基本のゲームをもとに、舞台、ステージ、難易度を自由に変更します。">

ここからは発展演習です。基本のゲームが完成し、時間に余裕がある場合に取り組んでください。すべて行う必要はありません。興味のある演習を選びましょう。

### 演習-発展1

<Exercise>

自分の`NIGHT ESCAPE`の舞台を決めてください。

```text
ゲームの場所：
ハンター：
脱出口：
目指す雰囲気：
```

例えば、次のような舞台があります。

- 停電した研究施設から、実験体を避けて脱出する
- 夜の迷路で、正体不明の怪物から逃げる
- 廃倉庫で、警備ロボットから逃げる
- 宇宙基地で、故障した作業ロボットから逃げる

今回作ったBlueprintと、標準アセット、Cube、ライトで表現できる内容にします。

<Hint>

「どこで、何から、どこへ逃げるか」を1つの場面として考えます。

</Hint>

<Answer>

次は舞台の一例です。

```text
ゲームの場所：停電した宇宙基地
ハンター：故障した警備ロボット
脱出口：青く光る緊急転送ゲート
目指す雰囲気：暗い通路で光を頼りに逃げる
```

4つの項目から同じ場面を想像でき、今回の機能で表現できる内容になっていれば完成です。

</Answer>

</Exercise>

### 演習-発展2

<Exercise>

決めた舞台に合わせて、`L_NightEscape`の壁、床、ライト、Actorの配置を作り替えてください。

次の条件を満たすステージを作ります。

- スタートから脱出口まで進める
- ハンターがプレイヤーを追跡できる
- 進むルートを選ぶ場所が1か所ある
- 舞台に合う見た目へ1か所以上変更している

<Hint>

最初に、スタート、警戒エリア、ハンターの巡回場所、脱出口を線でつなぎ、プレイヤーが通る順番を決めます。

</Hint>

<Answer>

例えば、次のような配置にできます。

```text
プレイヤー開始位置
        │
    警戒エリア
        │
    分かれ道
   ├──────┐
危険な近道  安全な遠回り
   └──────┤
        脱出口
```

巡回、追跡、捕獲、脱出が動き、プレイヤーが進む道を選べれば完成です。

</Answer>

</Exercise>

### 演習-発展3

<Exercise>

自分のステージを3回Playし、気になった点を1つ調整してください。

1. 脱出口まで逃げる
2. 別のルートを試す
3. ハンターに捕まってみる

次の中から、調整する項目を1つ選びます。

- ハンターの速さ
- `CaptureCollision`の大きさ
- 巡回地点
- 警戒エリア
- 脱出口
- 壁や通路

変更後にもう一度Playし、違いを確認してください。

<Hint>

一度に変える項目は1つだけにします。複数を同時に変えると、どの変更が結果に影響したのか分かりにくくなります。

</Hint>

<Answer>

例えば、ハンターがすぐ近づいてくる場合は、ハンターの速さを下げる、巡回場所を遠ざける、通路を広げるなどの調整ができます。

変更後に、狙ったとおりに難しさや追われ方が変われば完成です。

</Answer>

</Exercise>

### 演習-発展4

<Exercise>

これまでの授業で作った機能から、自分で追加できそうなものを1つ選び、`NIGHT ESCAPE`に追加してください。

例：

- 動く床を逃走経路に置く
- スイッチで新しい道を開く
- 物理オブジェクトを障害物にする
- Niagaraで脱出口を目立たせる
- 効果音で警戒や脱出を表現する
- UIで操作方法を表示する
- チェックポイントを追加する
- シーケンサーでゲーム開始時の演出を作る

追加後も、巡回、追跡、捕獲、脱出の基本機能が動くようにします。

<Hint>

追加する機能が、「逃げる」「見つかる」「脱出する」のどこを面白くするのか考えます。

</Hint>

<Answer>

例えば、宇宙基地を舞台にした場合は、Niagaraで転送ゲートを表現したり、スイッチで隔壁を開いたりできます。

追加した機能の効果をPlayで確認でき、基本のゲーム進行も動けば完成です。

</Answer>

</Exercise>

### 演習-発展5

<Exercise>

`BP_Hunter`をもう1体配置し、2体のハンターに異なる役割を持たせてください。

例えば、次のように役割を分けられます。

- 入口近くを巡回するハンターと、脱出口近くを巡回するハンター
- 近くにいる遅いハンターと、遠くにいる速いハンター
- 左側の道を守るハンターと、右側の道を守るハンター

ハンターごとに、2つの巡回地点と`BP_AlertZone`を用意します。各`BP_AlertZone`の`Hunter`には、対応するハンターを設定してください。

2体の役割の違いがPlay中に分かれば完成です。

<Hint>

2体の開始位置、巡回経路、速さ、警戒エリアのどれかを変えると、役割の違いが伝わりやすくなります。

</Hint>

<Answer>

例えば、1体を入口付近でゆっくり巡回させ、もう1体を脱出口付近で速く巡回させると、守る場所と危険度の違いが伝わります。

それぞれの警戒エリアが正しいハンターを動かし、2体の違いがプレイから分かれば完成です。

</Answer>

</Exercise>

</Section>
