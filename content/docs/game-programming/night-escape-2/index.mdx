このページでは、ハンターに捕まったときの再スタートと、起動装置で開く脱出口、ゲームクリアを追加します。

<Prerequisites>

- [NIGHT ESCAPE 1：警備中のハンターから逃げろ！](../night-escape-1/index.mdx)を完了している

</Prerequisites>

<Checkpoint title="このページで作るもの">

- 捕まると捕獲メッセージが表示され、レベルが最初から始まる
- 起動装置を作動させると、脱出口が使えるようになる
- 脱出口に入ると脱出メッセージが表示され、ゲームが停止する

</Checkpoint>

<Section title="捕まったら最初からやり直す" goal="ハンターに捕まるとメッセージが表示され、レベルが最初から始まるようにします。">

<Action title="捕獲範囲を追加する">

`BP_Hunter`に`Sphere Collision`を追加し、名前を`CaptureCollision`に変更します。

</Action>

<Action title="捕獲範囲を設定する">

`CaptureCollision`をハンターの体を囲む大きさへ調整し、次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="Overlapイベントを追加する">

`CaptureCollision`の`On Component Begin Overlap`を追加します。

</Action>

<Action title="プレイヤーか確認する">

`Other Actor`ピンから`Cast To BP_ThirdPersonCharacter`を追加します。

```text
CaptureCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

</Action>

<Action title="捕獲判定を無効にする">

Cast成功側から`CaptureCollision`の`Set Collision Enabled`を実行し、`New Type`を`No Collision`にします。

捕獲処理の重複を防ぐため、プレイヤーを捕まえた時点で判定を無効にします。

</Action>

<Action title="メッセージを表示する">

`Print String`を追加し、捕まったことが分かる文字を設定します。例として`In String`を`CAUGHT!`にし、`Duration`を`2.0`にします。`Text Color`は自由です。

</Action>

<Action title="再スタートまで待つ">

`Print String`のあとに`Delay`を接続し、`Duration`を`2.0`にします。

</Action>

<Action title="レベルを開き直す">

`Delay`のあとに`Open Level (by Object Reference)`を接続し、`Level`欄の一覧から`L_NightEscape`を選びます。

</Action>

完成した接続は次のようになります。

```text
Cast成功
    ↓
Set Collision Enabled
Target: CaptureCollision
New Type: No Collision
    ↓
Print String
In String: 捕まったことが分かる文字（例：CAUGHT!）
Duration: 2.0
Text Color: 自由
    ↓
Delay
Duration: 2.0
    ↓
Open Level (by Object Reference)
Level: L_NightEscape
```

<Action title="捕獲処理を保存する">

`BP_Hunter`をCompileしてSaveします。

</Action>

<Action title="捕まる距離を調整する">

Playし、見た目に合う距離で捕まるように`CaptureCollision`を調整します。

</Action>

<Section title="2つの起動装置で脱出口を開く" goal="左右の起動装置を両方作動させたときだけ、脱出口が使えるようにします。">

### 演習1

<Exercise>

`BP_EscapeSwitch`を左右の通路に1つずつ配置し、2つとも作動させたときだけ脱出口が開くようにしてください。

次の状態になれば完成です。

- 開始時は脱出口を使えない
- 片方の起動装置だけでは脱出口が開かない
- 2つの起動装置を作動させると脱出口が開く
- 各起動装置は1回だけ反応する
- 脱出口が開いたあとに入るとゲームクリアになる

<Hint>

まず、レベル上の`BP_EscapeSwitch`を複製し、左右の通路へ1つずつ配置します。2つとも、Detailsの`EscapeGate`に同じ`BP_EscapeGate`を割り当ててください。

</Hint>

<Hint>

`BP_EscapeGate`に、作動した起動装置の数を保存するInteger型の変数を作ります。`ActivateEscapeGate`が呼ばれるたびに値を1増やし、値が2になった場合だけ、脱出口を有効にする処理へ進めます。

</Hint>

<Answer>

`BP_EscapeGate`にInteger型の変数`ActivatedSwitchCount`を作り、初期値を`0`にします。

`ActivateEscapeGate`の先頭で、現在の`ActivatedSwitchCount`に`1`を足し、その結果を`ActivatedSwitchCount`へ保存します。続けて`ActivatedSwitchCount >= 2`を`Branch`の`Condition`へ接続します。

```text
ActivateEscapeGate
    ↓
ActivatedSwitchCount + 1
    ↓
Set ActivatedSwitchCount
    ↓
ActivatedSwitchCount >= 2
    ↓
Branch

True
    ↓
Set Collision Enabled
Target: EscapeCollision
New Type: Query Only
    ↓
Set Light Color
Target: EscapeLight
    ↓
Print String
In String: 脱出口が開いたことが分かる文字（例：EXIT OPEN!）
```

元からあった脱出口を有効にする処理は、`Branch`の`True`側へ移します。`False`側には何も接続しません。これにより、1つ目の起動装置では数だけが増え、2つ目が作動したときに脱出口が開きます。

レベルでは`BP_EscapeSwitch`を左右の通路に1つずつ置き、両方の`EscapeGate`に同じ`BP_EscapeGate`を割り当てます。各`BP_EscapeSwitch`は作動直後に自身の`SwitchCollision`を無効にするため、同じ起動装置で数が繰り返し増えることはありません。

Playして、1つ目を作動させても脱出口が起動前の色のままであること、2つ目を作動させるとメッセージが表示され、脱出口の色が変わることを確認します。その後、脱出口へ入ってゲームクリアになれば完成です。

</Answer>

</Exercise>

</Section>

<Section title="時間が余ったらゲームをアレンジする" goal="既習機能を使い、見た目や遊び方を発展させます。">

すべて行う必要はありません。興味のある演習を選びましょう。

### 演習-発展1

<Exercise>

動作確認のために`Print String`で表示している、次の3つをゲームに合う演出へ置き換えてください。

- 捕まったときのメッセージ（例：`CAUGHT!`）
- 脱出口が開いたときのメッセージ（例：`EXIT OPEN!`）
- 脱出したときのメッセージ（例：`ESCAPED!`）

UI、音、Widget Animation、Niagaraなど、これまでに学んだ機能を使えます。

次の状態になれば完成です。

- 3つすべての`Print String`が演出へ置き換わっている
- 捕獲、脱出口の起動、脱出を見分けられる
- 捕獲演出のあとにレベルが再スタートする
- 脱出口の起動演出は一定時間後に消え、そのままゲームを続けられる
- 脱出演出のあとにゲームが停止する

<Hint>

まず、`Print String`がある3か所を確認します。

- `BP_Hunter`の捕獲処理
- `BP_EscapeGate`の`ActivateEscapeGate`
- `BP_EscapeGate`の脱出処理

一度にすべて変更せず、最初に1か所だけWidgetや音へ置き換えて、元のゲーム進行が動くか確認してください。

</Hint>

<Hint>

Widgetを使う場合は、`Create Widget`の`Return Value`を`Add to Viewport`へ接続します。

- 捕獲：演出のあとに既存の`Delay`と`Open Level`へ進む
- 脱出口の起動：演出を表示し、`Delay`のあとに`Remove from Parent`で消す
- 脱出：演出を表示してから`Set Game Paused`へ進む

演出が正しく動くことを確認してから、対応する`Print String`を削除してください。

</Hint>

<Answer>

一例として、次の3つのWidgetを作ります。

| Widget | 表示内容の例 | 役割 |
| ------ | ------------ | ---- |
| `WBP_Caught` | 赤い背景と`CAUGHT!` | 捕まったことを伝える |
| `WBP_ExitOpen` | 緑色の`EXIT OPEN!` | 脱出口が使えるようになったことを伝える |
| `WBP_Escaped` | 青または緑の背景と`ESCAPED!` | 脱出成功を伝える |

3つを同じ見た目にせず、色、文字の大きさ、表示位置、音などを変えると、何が起きたのか判断しやすくなります。

### 捕獲演出

`BP_Hunter`の捕獲処理で、`Print String`を`WBP_Caught`の表示へ置き換えます。

```text
捕獲判定を無効にする
    ↓
Create Widget: WBP_Caught
    ↓
Add to Viewport
    ↓
Delay: 2.0
    ↓
Open Level: L_NightEscape
```

捕獲音を使う場合は、Widgetを表示するときに再生します。`Delay`と`Open Level`は残します。これらを削除すると、演出を表示してもレベルが再スタートしません。

### 脱出口の起動演出

`BP_EscapeGate`の`ActivateEscapeGate`で、`Print String`を`WBP_ExitOpen`の表示へ置き換えます。

```text
Set Collision Enabled
    ↓
Set Light Color
    ↓
Create Widget: WBP_ExitOpen
    ↓
Add to Viewport
    ↓
Delay: 2.0
    ↓
Remove from Parent
```

`Create Widget`の`Return Value`を、`Add to Viewport`と`Remove from Parent`の両方で使います。2秒後にWidgetだけを消すため、脱出口が開いた状態とゲームの進行はそのまま残ります。

### 脱出演出

`BP_EscapeGate`の脱出処理で、`Print String`を`WBP_Escaped`の表示へ置き換えます。

```text
脱出判定を無効にする
    ↓
Create Widget: WBP_Escaped
    ↓
Add to Viewport
    ↓
Set Game Paused
```

脱出音を使う場合は、`Set Game Paused`より前に再生します。脱出後はゲームを停止するため、結果画面は自分で消す必要はありません。

最後にPlayし、次の順で確認します。

1. ハンターに捕まると捕獲演出が表示され、2秒後に再スタートする
2. 2つの起動装置を作動させると起動演出が表示され、2秒後に表示だけが消える
3. 脱出口へ入ると脱出演出が表示され、ゲームが停止する

3つの`Print String`を削除した状態で、3つの出来事が区別でき、元のゲーム進行も動けば完成です。

</Answer>

</Exercise>
