title: NIGHT ESCAPE 2：脱出口まで逃げ切れ！
---

このページでは、ハンターに捕まったときの再スタートと、脱出口に入ったときのゲームクリアを追加します。

<Prerequisites>

- [NIGHT ESCAPE 1：警備中のハンターから逃げろ！](../night-escape-1/index.mdx)を完了している

</Prerequisites>

<Checkpoint title="このページで作るもの">

- 捕まると`CAUGHT!`が表示され、レベルが最初から始まる
- 脱出口に入ると`ESCAPED!`が表示され、ゲームが停止する

</Checkpoint>

---

<Section title="捕まったら最初からやり直す" goal="ハンターに捕まるとメッセージが表示され、レベルが最初から始まるようにします。">

<Action title="捕獲範囲を追加する">

`BP_Hunter`に`Sphere Collision`を追加し、名前を`CaptureCollision`に変更します。

</Action>

<Action title="捕獲範囲を設定する">

`CaptureCollision`をハンターの体を囲む大きさへ調整し、次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="Overlapイベントを追加する">

`CaptureCollision`の`On Component Begin Overlap`を追加します。

</Action>

<Action title="プレイヤーか確認する">

`Other Actor`ピンから`Cast To BP_ThirdPersonCharacter`を追加します。

```text
CaptureCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
```

</Action>

<Action title="捕獲判定を無効にする">

Cast成功側から`CaptureCollision`の`Set Collision Enabled`を実行し、`New Type`を`No Collision`にします。

捕獲処理の重複を防ぐため、プレイヤーを捕まえた時点で判定を無効にします。

</Action>

<Action title="メッセージを表示する">

`Print String`を追加し、`In String`を`CAUGHT!`、`Duration`を`0.5`にします。

</Action>

<Action title="再スタートまで待つ">

`Print String`のあとに`Delay`を接続し、`Duration`を`0.5`にします。

</Action>

<Action title="レベルを開き直す">

`Delay`のあとに`Open Level (by Object Reference)`を接続し、`Level`欄の一覧から`L_NightEscape`を選びます。

</Action>

完成した接続は次のようになります。

```text
Cast成功
    ↓
Set Collision Enabled
Target: CaptureCollision
New Type: No Collision
    ↓
Print String
In String: CAUGHT!
Duration: 0.5
    ↓
Delay
Duration: 0.5
    ↓
Open Level (by Object Reference)
Level: L_NightEscape
```

<Action title="捕獲処理を保存する">

`BP_Hunter`をCompileしてSaveします。

</Action>

<Action title="捕まる距離を調整する">

Playし、見た目に合う距離で捕まるように`CaptureCollision`を調整します。

</Action>

<Verify title="捕獲と再スタートを確認する">

- 捕まると`CAUGHT!`が表示される
- レベルが最初から始まり、ハンターが再び巡回する

</Verify>

<Recovery title="近づいてもCAUGHT!が表示されないとき">

- `CaptureCollision`の`Generate Overlap Events`が有効か確認する
- `Other Actor`がCastの`Object`につながっているか確認する
- 体が触れても反応しない場合は、`CaptureCollision`を大きくする
- `BP_Hunter`をCompileしてSaveする

</Recovery>

<Recovery title="離れているのに捕まるとき">

- `CaptureCollision`を小さくして、もう一度Playする

</Recovery>

<Recovery title="CAUGHT!は表示されるが再スタートしないとき">

- `Print String`のあとに`Delay`がつながっているか確認する
- `Delay`のあとに`Open Level (by Object Reference)`がつながっているか確認する
- `Level`に`L_NightEscape`が設定されているか確認する
- 一覧に`L_NightEscape`が表示されない場合は、レベルをSaveしてから選び直す

</Recovery>

<Checkpoint title="理解度確認1">

`Print String`のあとに`Delay`を入れるのはなぜですか。

<details>
<summary>ヒント</summary>

`Open Level`を実行すると、画面がすぐに開始時の状態へ変わります。

</details>

<details>
<summary>答え</summary>

レベルを開き直す前に、`CAUGHT!`を画面に表示する時間を作るためです。

</details>

</Checkpoint>

</Section>

<Section title="脱出口を作る" goal="プレイヤーが中に入れる、光る脱出口を作ります。">

<Action title="脱出口のBlueprintを作る">

`Content/NightEscape/Blueprints`に、`Actor`を親にした`BP_EscapeGate`を作ります。

</Action>

<Action title="脱出口のコンポーネントを追加する">

次のコンポーネントを追加します。

| コンポーネント  | 名前              |
| --------------- | ----------------- |
| `Cube`          | `GateMesh`        |
| `Box Collision` | `EscapeCollision` |
| `Point Light`   | `EscapeLight`     |

</Action>

<Action title="脱出口の見た目を作る">

`GateMesh`を縦長の入口の形にし、`Collision Presets`を`NoCollision`にします。

</Action>

<Action title="脱出判定を設定する">

`EscapeCollision`をプレイヤーが入れる大きさにし、次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="ライトを調整する">

`EscapeLight`を、脱出口の位置が分かり、ハンターのライトと見分けられる色と明るさにします。

</Action>

<Action title="脱出口を保存する">

`BP_EscapeGate`をCompileしてSaveします。

</Action>

<Action title="脱出口をレベルに配置する">

`BP_EscapeGate`を、ステージ上側の中央に空けておいた脱出口予定地へ配置します。

</Action>

<Verify title="脱出口を確認する">

- 脱出口の位置が分かる
- プレイヤーが`EscapeCollision`の中へ入れる
- ハンターのライトと見分けられる

</Verify>

</Section>

<Section title="脱出口に入ったらゲームクリアにする" goal="脱出口に入るとメッセージが表示され、ゲームが停止するようにします。">

<Action title="Overlapイベントを追加する">

`BP_EscapeGate`で`EscapeCollision`の`On Component Begin Overlap`を追加します。

</Action>

<Action title="プレイヤーか確認する">

捕獲処理と同じように、`Other Actor`ピンから`Cast To BP_ThirdPersonCharacter`を追加します。

</Action>

<Action title="脱出判定を無効にする">

Cast成功側から`EscapeCollision`の`Set Collision Enabled`を実行し、`New Type`を`No Collision`にします。

</Action>

<Action title="メッセージを表示する">

`Print String`を追加し、`In String`を`ESCAPED!`、`Duration`を`10`にします。

</Action>

<Action title="ゲームを停止する">

`Print String`のあとに`Set Game Paused`を接続し、`Paused`を有効にします。

</Action>

完成した接続は次のようになります。

```text
Cast成功
    ↓
Set Collision Enabled
Target: EscapeCollision
New Type: No Collision
    ↓
Print String
In String: ESCAPED!
Duration: 10
    ↓
Set Game Paused
Paused: true
```

<Action title="ゲームクリア処理を保存する">

`BP_EscapeGate`をCompileしてSaveします。

</Action>

<Verify title="ゲームクリアを確認する">

- 脱出口へ入ると`ESCAPED!`が表示される
- プレイヤーとハンターの動きが止まる

</Verify>

<Recovery title="脱出口に入ってもESCAPED!が表示されないとき">

- `EscapeCollision`の`Generate Overlap Events`が有効か確認する
- `Other Actor`がCastの`Object`につながっているか確認する
- Cast成功側から`Print String`まで実行ピンがつながっているか確認する
- `BP_EscapeGate`をCompileしてSaveする

</Recovery>

<Recovery title="ESCAPED!は表示されるがゲームが停止しないとき">

- `Print String`のあとに`Set Game Paused`がつながっているか確認する
- `Set Game Paused`の`Paused`が有効になっているか確認する
- Cast成功側から`Set Game Paused`まで実行ピンが途切れていないか確認する

</Recovery>

<Checkpoint title="理解度確認2">

脱出処理の最初に`EscapeCollision`を無効にするのはなぜですか。

<Section title="時間が余ったらゲームをアレンジする" goal="既習機能を使い、見た目や遊び方を発展させます。">

すべて行う必要はありません。興味のある演習を選びましょう。

### 演習-発展1

<Exercise>

ゲームの舞台を決め、その舞台に合う壁、通路、ハンター、脱出口、ライトへ作り替えてください。

基本の巡回、追跡、捕獲、脱出が動いたまま、何を表したステージなのか見た目から分かれば完成です。

<Hint>

「どこで、何から、どこへ逃げるか」を1つの場面として決めます。

</Hint>

<Answer>

例えば、舞台を「停電した宇宙基地」にします。

```text
場所：停電した宇宙基地
ハンター：赤い警告灯を点灯する警備ロボット
脱出口：青く光る緊急転送ゲート
雰囲気：暗い通路で光を頼りに逃げる
```

壁や通路には金属製のアセットや暗いマテリアルを使い、巡回中のハンターは通常色、追跡時は赤色、脱出口は青色にします。ハンターと脱出口の光を見分けられるため、暗いステージでも進む方向が分かります。

見た目を変更したあとも、Navigation Meshが床全体につながり、警戒エリア、巡回、追跡、捕獲、脱出がすべて動くことを確認します。舞台の説明とステージの見た目が一致していれば完成です。

</Answer>

</Exercise>

### 演習-発展2

<Exercise>

これまでの授業で作った機能から1つ選び、逃走や脱出が面白くなるように追加してください。

例：

- 動く床
- スイッチ
- 物理オブジェクト
- Niagara
- 効果音
- UI
- チェックポイント
- シーケンサー

追加後も、巡回、追跡、捕獲、脱出がすべて動けば完成です。

<Hint>

追加する機能が、「逃げる」「見つかる」「脱出する」のどこを変えるのかを先に決めます。

</Hint>

<Answer>

一例として、スイッチを押すと近道が開く仕掛けを追加します。既習のスイッチBlueprintから、通路をふさいでいる扉や床のCustom Eventを呼び出し、プレイヤーが操作したときだけ近道を通れるようにします。

ハンターに追われながらスイッチへ向かう必要がある位置へ配置すると、単なる装飾ではなく逃走中の選択になります。近道が開く前にも脱出口へ到達できる通常ルートを残しておくと、仕掛けが動かない場合でもゲーム全体を確認できます。

Playして、スイッチを使わない場合と使った場合で逃げ方が変わること、追加した扉や床がNavigation Meshやハンターの移動を壊していないこと、捕獲と脱出が引き続き動くことを確認します。別の既習機能を選んだ場合も、ゲーム中の役割と動作確認が説明できれば完成です。

</Answer>

</Exercise>

### 演習-発展3

<Exercise>

`BP_Hunter`をもう1体配置し、2体に違う役割を持たせてください。

ハンターごとに巡回地点と警戒エリアを用意し、Playしたときに役割の違いが分かれば完成です。

<Hint>

開始位置、巡回経路、速度、警戒エリアのどれかを変えると、役割を分けやすくなります。

</Hint>

<Answer>

例えば、1体目は入口に近い場所をゆっくり巡回し、2体目は脱出口に近い場所を速く巡回させます。それぞれに別の`PatrolPointA`と`PatrolPointB`を割り当て、巡回経路が重なりすぎないようにします。

警戒エリアもハンターごとに用意します。各`BP_AlertZone`の`Hunter`には、その警戒エリアで追跡を始める`BP_Hunter`を割り当てます。同じハンターを両方へ割り当てると、もう1体が追跡を始めないため注意します。

Playして、警戒エリアに入る前は2体が別の場所を巡回し、それぞれの警戒エリアへ入ると対応するハンターが追跡を始めることを確認します。2体の開始位置、経路、速度の違いがプレイ中に分かれば完成です。

</Answer>

</Exercise>

</Section>
