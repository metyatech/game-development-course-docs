---
title: NIGHT ESCAPE 1：警備中のハンターから逃げろ！
---

このページでは、2つの地点を巡回し、プレイヤーが警戒エリアに入ると追いかけてくるハンターを作ります。

<Prerequisites>

- Unreal Engine 5でThird PersonのBlueprintプロジェクトを開ける
- Actorをレベルに配置し、移動・回転・拡大縮小を操作できる
- Blueprintを複製し、名前を変更できる
- Event Graphにノードを配置し、ピンを接続できる
- BlueprintをCompile、Saveできる

</Prerequisites>

<Checkpoint title="このページで作るもの">

- ハンターが2つの地点を往復する
- プレイヤーが警戒エリアに入ると、ハンターのライトが赤くなる
- ハンターが巡回をやめ、プレイヤーを追いかける
- ハンターが壁を回り込んで移動する

</Checkpoint>

<Section title="作業用のレベルを用意する" goal="テンプレートのレベルを複製し、NIGHT ESCAPE用の作業場所を用意します。">

<Action title="フォルダを作る">

コンテンツドロワーの`Content`直下に、`NightEscape`フォルダを作ります。

その中に、`Blueprints`フォルダと`Maps`フォルダを作ります。

```text
Content/
└── NightEscape/
    ├── Blueprints/
    └── Maps/
```

</Action>

<Action title="レベルを別名で保存する">

メニューから`File`→`Save Current Level As...`を選びます。

保存先に`Content/NightEscape/Maps`を選び、レベル名を`L_NightEscape`にします。

保存後は、`L_NightEscape`を開いたまま作業を進めます。テンプレートの元のレベルは、そのままプロジェクト内に残ります。

</Action>

<Verify title="レベルを確認する">

- `Maps`フォルダに`L_NightEscape`がある
- エディタ上部に`L_NightEscape`と表示されている
- Playするとプレイヤーキャラクターを操作できる

</Verify>

</Section>

---

<Section title="動作確認用のステージを作る" goal="巡回と追跡を試せる小さなステージを作ります。">

まずは、ハンターの動きを確認するための小さなステージを作ります。見た目はあとで調整するため、ここでは通路と壁だけを用意します。

完成形は、入口から奥へ進む途中に曲がり角があり、その奥をハンターが巡回する形です。

```text
             巡回地点B
                 │
                 │
                 └──── 巡回地点A
                       ┌───────┐
                       │  壁   │
                       └───────┘

                  警戒エリア

                    プレイヤー
```

<Action title="通路と壁を配置する">

Third Personテンプレートの床を使い、プレイヤーが開始する入口を決めます。

入口から少し離れた場所に、ハンターが巡回できる空間を作ります。入口と奥の間には、Cubeで回り込める壁を1つ置きます。

プレイヤーとハンターが壁の横を通れるように、通路の幅を確保してください。

</Action>

<Verify title="ステージの形を確認する">

- 入口から奥まで歩いて移動できる
- 入口と奥の間に、回り込める壁がある
- プレイヤーが入口側へ戻って逃げられる

</Verify>

</Section>

<Section title="ハンターが移動できる道を作る" goal="Navigation Meshを使って、ハンターが移動できる道を作ります。">

<Concept title="ハンターが移動できる範囲">

ハンターは、Navigation Meshを使って目的地までの道を探します。

`P`キーを押すと、ハンターが移動できる床が緑色で表示されます。

</Concept>

<Action title="Nav Mesh Bounds Volumeを配置する">

Place Actorsで`Nav Mesh Bounds Volume`を検索し、レベルに配置します。

Volumeを移動・拡大し、今回使う床全体を覆います。床の高さもVolumeの中に入るように調整してください。

</Action>

<Action title="移動できる道を表示する">

ビューポートをクリックしてから`P`キーを押します。

床の上に、緑色の範囲が表示されます。

</Action>

<Verify title="Navigation Meshを確認する">

- 使用する床が緑色で表示される
- 入口から奥まで、壁を回り込む緑色の道がつながっている

</Verify>

<Recovery title="緑色の範囲が表示されない">

- `Nav Mesh Bounds Volume`が床全体を覆っているか確認する
- Volumeの高さに床が含まれているか確認する
- 床のCollisionが有効か確認する
- ビューポートを選択してから`P`キーを押す

</Recovery>

<Checkpoint title="理解度確認1">

`P`キーを押したときに表示される緑色の範囲は、何を表していますか。

<details>
<summary>ヒント</summary>

ハンターが目的地まで移動するときに使う範囲です。

</details>

<details>
<summary>答え</summary>

ハンターが道を探して移動できる範囲です。

</details>

</Checkpoint>

</Section>

<Section title="ハンターを作る" goal="プレイヤーキャラクターを複製し、AIで動くハンターを作ります。">

<Action title="BP_Hunterを作る">

コンテンツドロワーで`BP_ThirdPersonCharacter`を探します。

`BP_ThirdPersonCharacter`を複製して`Content/NightEscape/Blueprints`へ移動し、名前を`BP_Hunter`に変更します。

</Action>

<Action title="プレイヤー用の入力処理を削除する">

`BP_Hunter`を開きます。

Event Graphにある`Move`、`Look`、`Jump`の入力イベントと、そこにつながっているノードを削除します。

ハンターの移動は、このあと作るAIの処理から実行します。

</Action>

<Action title="AIで操作できるようにする">

`BP_Hunter`のClass Defaultsを開き、次の項目を設定します。

| 項目                  | 値                           |
| --------------------- | ---------------------------- |
| `AI Controller Class` | `AIController`               |
| `Auto Possess AI`     | `Placed in World or Spawned` |

これで、レベルに配置した`BP_Hunter`をAIControllerが操作できるようになります。

</Action>

<Action title="ライトと移動速度を設定する">

Componentsパネルで`Point Light`を追加し、名前を`HunterLight`に変更します。

`HunterLight`をハンターの頭や胸の近くへ移動します。巡回中の色は、白、黄色、青などから見やすいものを選びます。

次に、Componentsパネルで`Character Movement`を選びます。Detailsにある`Max Walk Speed`で、ハンターの移動速度を変更できます。ここでは、動きを確認しやすい速さにしておきます。

</Action>

<Action title="レベルに配置する">

`BP_Hunter`をステージの奥に配置します。

ハンターの足が床に接するように、高さを調整します。

</Action>

<Verify title="ハンターを確認する">

Playして、次を確認します。

- プレイヤーとは別のキャラクターが奥にいる
- `HunterLight`の光でハンターの位置が分かる
- ハンターが配置した場所に立っている

</Verify>

</Section>

<Section title="ハンターを巡回させる" goal="2つの目的地を設定し、ハンターを往復させます。">

<Concept title="巡回地点に使うTarget Point">

Target Pointは、レベル内の位置を表すActorです。

今回は、ハンターが巡回するときの目的地として2つ使います。

</Concept>

<Action title="巡回地点を2つ置く">

Place Actorsで`Target Point`を検索し、ステージの奥に2つ配置します。

World Outlinerで、名前を`PatrolPointA`と`PatrolPointB`に変更します。

2つの地点を離して置き、間に壁や曲がり角が入るようにします。どちらも緑色のNavigation Mesh上に置いてください。

</Action>

<Action title="ハンターに巡回地点を設定する">

`BP_Hunter`に、次の変数を作ります。

| 変数名         | 型                              |
| -------------- | ------------------------------- |
| `PatrolPointA` | `Target Point Object Reference` |
| `PatrolPointB` | `Target Point Object Reference` |

2つの変数で、Instance Editableを有効にします。

さらに、Boolean型の変数`IsChasingPlayer`を作ります。初期値は`false`です。

`IsChasingPlayer`には、ハンターがプレイヤーを追跡中かどうかを記録します。

- `false`：巡回する
- `true`：プレイヤーを追いかける

Compileしてレベルへ戻り、レベル上の`BP_Hunter`を選択します。

Detailsで、変数に次のActorを設定します。

| 変数           | 設定するActor            |
| -------------- | ------------------------ |
| `PatrolPointA` | レベル上の`PatrolPointA` |
| `PatrolPointB` | レベル上の`PatrolPointB` |

</Action>

<Action title="2つの巡回イベントを作る">

`BP_Hunter`のEvent Graphに、Custom Event`MoveToPatrolPointA`を作ります。

`Branch`と`AI MoveTo`を追加し、次のように接続します。

```text
MoveToPatrolPointA
    ↓
Branch
Condition: IsChasingPlayer

False
    ↓
AI MoveTo
Pawn: Self
Target Actor: PatrolPointA
Acceptance Radius: 50
```

`IsChasingPlayer`が`false`のときだけ、ハンターは`PatrolPointA`へ向かいます。

同じように、Custom Event`MoveToPatrolPointB`を作ります。

```text
MoveToPatrolPointB
    ↓
Branch
Condition: IsChasingPlayer

False
    ↓
AI MoveTo
Pawn: Self
Target Actor: PatrolPointB
Acceptance Radius: 50
```

`MoveToPatrolPointA`の`AI MoveTo`では、`On Success`から`MoveToPatrolPointB`を呼び出します。

`MoveToPatrolPointB`の`AI MoveTo`では、`On Success`から`MoveToPatrolPointA`を呼び出します。

```text
PatrolPointAへ到着
    ↓
MoveToPatrolPointB
    ↓
PatrolPointBへ到着
    ↓
MoveToPatrolPointA
```

</Action>

<Action title="ゲーム開始時に巡回を始める">

`Event BeginPlay`から`MoveToPatrolPointA`を呼び出します。

```text
Event BeginPlay
    ↓
MoveToPatrolPointA
```

CompileしてSaveします。

</Action>

<Verify title="巡回を確認する">

Playして、次を確認します。

1. ハンターが`PatrolPointA`へ向かう
2. 到着すると`PatrolPointB`へ向かう
3. `PatrolPointB`へ到着すると、再び`PatrolPointA`へ向かう
4. 壁や曲がり角に沿って移動する

</Verify>

目的地まで移動できない場合、`AI MoveTo`は`On Fail`になります。今回は`On Fail`から次の巡回を始めないため、ハンターはその場で止まります。

<Recovery title="ハンターがまったく動かない">

- `BP_Hunter`と2つの巡回地点がNavigation Mesh上にあるか確認する
- レベル上の`BP_Hunter`に、2つの巡回地点が設定されているか確認する
- `Event BeginPlay`から`MoveToPatrolPointA`を呼び出しているか確認する
- `AI MoveTo`の`Pawn`に`Self`を接続しているか確認する

</Recovery>

<Recovery title="片方の巡回地点で止まる">

- 止まった側の`AI MoveTo`の`On Success`が、反対側の巡回イベントにつながっているか確認する
- 次の巡回地点までNavigation Meshがつながっているか確認する
- `Target Actor`に正しい巡回地点の変数を接続しているか確認する

</Recovery>

<Checkpoint title="理解度確認2">

`PatrolPointA`がNavigation Meshの外にあると、ハンターはどうなりますか。

<details>
<summary>ヒント</summary>

ハンターは、緑色の範囲の中で目的地までの道を探します。

</details>

<details>
<summary>答え</summary>

ハンターは`PatrolPointA`へ到達できません。`AI MoveTo`が失敗し、`On Success`から次の巡回イベントを呼び出せないため、そこで止まります。

</details>

</Checkpoint>

</Section>

<Section title="警戒エリアに入ったら追跡を始める" goal="プレイヤーが警戒エリアに入ると、ハンターが追いかける処理を作ります。">

<Action title="BP_AlertZoneを作る">

`Content/NightEscape/Blueprints`に、`Actor`を親にしたBlueprintを作り、名前を`BP_AlertZone`にします。

`Box Collision`を追加し、名前を`AlertCollision`に変更します。

`AlertCollision`を次のように設定します。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

Boxの大きさは、プレイヤーが通る通路の幅に合わせます。

</Action>

<Action title="警戒エリアにハンターを設定する">

`BP_AlertZone`に、`BP_Hunter Object Reference`型の変数`Hunter`を作ります。

`Hunter`でInstance Editableを有効にします。

Compileしてレベルへ戻り、`BP_AlertZone`を通路を横切る位置に配置します。

レベル上の`BP_AlertZone`を選択し、Detailsの`Hunter`にレベル上の`BP_Hunter`を設定します。

</Action>

<Action title="ハンターに追跡処理を作る">

`BP_Hunter`に、Custom Event`StartChasingPlayer`を作ります。

イベントの先頭で、`IsChasingPlayer`を`true`にします。

```text
StartChasingPlayer
    ↓
Set IsChasingPlayer
Value: true
```

Componentsパネルから`HunterLight`をEvent Graphにドラッグし、`Set Light Color`を追加します。`New Light Color`は赤色にします。

```text
Set IsChasingPlayer
    ↓
Set Light Color
Target: HunterLight
New Light Color: 赤
```

続けて、`AI MoveTo`と`Get Player Character`を追加します。

| `AI MoveTo`のピン | 接続・値                              |
| ----------------- | ------------------------------------ |
| `Pawn`            | `Self`                               |
| `Target Actor`    | `Get Player Character`の`Return Value` |
| `Acceptance Radius` | `80`                               |
| `Stop on Overlap` | 有効                                 |

実行ピンは、`Set Light Color`から`AI MoveTo`へ接続します。

`Target Actor`にプレイヤーキャラクターを指定すると、ハンターは移動するプレイヤーへ向かい続けます。

</Action>

<Action title="プレイヤーが入ったら追跡を始める">

`BP_AlertZone`のEvent Graphで、`AlertCollision`の`On Component Begin Overlap`を追加します。

`Other Actor`を`Cast To BP_ThirdPersonCharacter`の`Object`に接続します。

Cast成功側から、`Hunter`変数の`StartChasingPlayer`を呼び出します。

```text
AlertCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
    ↓
Hunter
StartChasingPlayer
```

</Action>

<Action title="追跡処理が繰り返されないようにする">

`StartChasingPlayer`を呼び出したあとに、`AlertCollision`の`Set Collision Enabled`を実行します。

`New Type`を`No Collision`にします。

```text
StartChasingPlayer
    ↓
Set Collision Enabled
Target: AlertCollision
New Type: No Collision
```

これで、警戒エリアに入ったときの処理は一度だけ実行されます。

</Action>

<Verify title="巡回から追跡へ変わるか確認する">

Playして、次を確認します。

1. ハンターが2つの地点を巡回する
2. プレイヤーが警戒エリアに入る
3. ハンターのライトが赤くなる
4. ハンターがプレイヤーを追いかける
5. ハンターが壁や曲がり角を回り込む

</Verify>

<Recovery title="警戒エリアに入っても追いかけてこない">

- `BP_AlertZone`の`Hunter`に、レベル上の`BP_Hunter`が設定されているか確認する
- `AlertCollision`のOverlapが有効か確認する
- `Other Actor`をCastの`Object`に接続しているか確認する
- Cast成功側から`StartChasingPlayer`を呼び出しているか確認する
- `StartChasingPlayer`の`AI MoveTo`で、`Pawn`に`Self`、`Target Actor`にプレイヤーを接続しているか確認する

</Recovery>

<Checkpoint title="理解度確認3">

追跡を始めるとき、最初に`IsChasingPlayer`を`true`にするのはなぜですか。

<details>
<summary>ヒント</summary>

巡回イベントの`Branch`では、`IsChasingPlayer`を確認しています。

</details>

<details>
<summary>答え</summary>

巡回イベントが再び呼び出されても、`Branch`のFalse側へ進まず、巡回用の`AI MoveTo`を実行しないようにするためです。

</details>

</Checkpoint>

</Section>

<Section title="動きを確認して調整する" goal="設定や配置を1か所変え、追われ方の変化を確かめます。">

### 演習1

<Exercise>

壁、巡回地点、警戒エリア、ハンターの速さから1つを選び、設定や配置を変更してください。

変更後も巡回と追跡が動き、変更前と比べてハンターの動きや追われ方が変われば完成です。

<Hint>

ハンターが止まった場合は、`P`キーでNavigation Meshを表示し、移動する道が途切れていないか確認します。

</Hint>

<Answer>

例えば、壁を1つ追加すると、ハンターがプレイヤーへまっすぐ進まず、壁を回り込んで追いかけるようになります。

巡回と追跡が動いたまま、変更前との違いをPlayで確認できれば完成です。

</Answer>

</Exercise>

<Checkpoint title="完成チェック">

- Navigation Meshが入口から奥までつながっている
- ハンターが2つの地点を往復する
- 警戒エリアに入るとライトが赤くなる
- ハンターがプレイヤーを追いかける
- ハンターが壁を回り込んで移動する
- 配置または速度を1か所調整した

</Checkpoint>

ここからは、時間に余裕がある場合に取り組みます。

### 演習-発展1

<Exercise>

警戒エリアに入る前からハンターの存在が伝わるように、壁、巡回地点、`HunterLight`の配置を調整してください。

例えば、次のような見せ方があります。

- 壁越しにライトの光が見える
- 曲がり角からハンターが一瞬見える
- 長い通路の奥をハンターが横切る
- 警戒エリアに入った直後、ハンターが正面を向く

警戒エリアに入る前と入ったあとで、ハンターの見え方が変われば完成です。

<Hint>

Blueprintを変更する前に、壁、巡回地点、ライトの位置だけで見せ方を変えられないか試します。

</Hint>

<Answer>

壁の向こうにハンターを配置し、ライトだけが通路から見えるようにすると、姿が見える前からハンターの存在を伝えられます。

その後、警戒エリアに入ってライトが赤くなり、ハンターが追いかけてくると、巡回から追跡への変化も分かりやすくなります。

</Answer>

</Exercise>
