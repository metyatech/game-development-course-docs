---
title: NIGHT ESCAPE 1：警備中のハンターから逃げろ！
---

このページでは、2つの地点を巡回し、プレイヤーが警戒エリアに入ると追いかけてくるハンターを作ります。

<Prerequisites>

- Unreal Engine 5でThird PersonのBlueprintプロジェクトを開いている

</Prerequisites>

<Checkpoint title="このページで作るもの">

- ハンターが2つの地点を往復する
- プレイヤーが警戒エリアに入ると、ハンターのライトが赤くなる
- ハンターが巡回をやめ、プレイヤーを追いかける
- ハンターが壁を回り込んで移動する

</Checkpoint>

<Section title="作業用のレベルを用意する" goal="NIGHT ESCAPE用のフォルダとレベルを用意します。">

<Action title="フォルダを作る">

コンテンツドロワーの`Content`直下に`NightEscape`フォルダを作り、その中に`Blueprints`と`Maps`を作ります。

```text
Content/
└── NightEscape/
    ├── Blueprints/
    └── Maps/
```

</Action>

<Action title="レベルを別名で保存する">

メニューから`File`→`Save Current Level As...`を選び、`Content/NightEscape/Maps`へ`L_NightEscape`として保存します。

別名で保存するため、元のテンプレートレベルも残ります。

</Action>

<Verify title="作業用のレベルを確認する">

- `Maps`フォルダに`L_NightEscape`がある
- エディタ上部に`L_NightEscape`と表示されている

</Verify>

</Section>

<Section title="ハンターが移動できる道を作る" goal="Navigation Meshを使って、ハンターが移動できる範囲を作ります。">

<Concept title="ハンターが道を探せる範囲">

ハンターは、Navigation Meshを使って目的地までの道を探します。

`P`キーを押すと、ハンターが移動できる床が緑色で表示されます。

</Concept>

<Action title="Nav Mesh Bounds Volumeを配置する">

Place Actorsで`Nav Mesh Bounds Volume`を検索し、今回使う床全体を覆うように配置します。

</Action>

<Action title="移動できる範囲を表示する">

ビューポートをクリックしてから`P`キーを押します。

</Action>

<Verify title="Navigation Meshを確認する">

- 床がすべて緑色で表示されている

</Verify>

<Recovery title="緑色の範囲が表示されないとき">

- `Nav Mesh Bounds Volume`がレベルにあるか確認する
- 床の上面がVolumeの中に入っているか確認する
- 床のCollisionが有効か確認する
- ビューポートを選択してから`P`キーを押す

</Recovery>

`P`キーで表示される緑色の範囲は、何を表していますか。

</details>

</Checkpoint>

<Section title="ハンターを作る" goal="プレイヤーキャラクターをもとに、自動で動くハンターを作ります。">

<Action title="ハンター用のBlueprintを作る">

`BP_ThirdPersonCharacter`を複製して`Content/NightEscape/Blueprints`へ移動し、名前を`BP_Hunter`に変更します。

</Action>

<Action title="プレイヤー用の入力処理を削除する">

`BP_Hunter`のEvent Graphにある`Move`、`Look`、`Jump`の入力イベントと、そこにつながっているノードを削除します。

</Action>

<Action title="AIの設定をする">

`BP_Hunter`のClass Defaultsで、次の項目を設定します。

| 項目                  | 値                |
| --------------------- | ----------------- |
| `AI Controller Class` | `AIController`    |
| `Auto Possess AI`     | `Placed in World` |

</Action>

<Action title="ライトを追加する">

Componentsパネルで`Point Light`を追加して`HunterLight`に変更し、ハンターの頭や胸の近くへ移動します。

</Action>

<Action title="ハンターをレベルに配置する">

`BP_Hunter`をステージ上側の中央へ配置し、足が床に接するように高さを調整します。

</Action>

<Action title="ライトを調整する">

`HunterLight`の`Light Color`と`Intensity`を調整し、`PlayerStart`からハンターの位置が分かる状態にします。

</Action>

<Action title="ハンターを保存する">

`BP_Hunter`をCompileしてSaveします。

</Action>

<Verify title="ハンターの初期状態を確認する">

- `PlayerStart`からハンターが見える
- `HunterLight`でハンターの位置が分かる

</Verify>

</Section>

<Section title="ハンターを巡回させる" goal="2つの巡回地点を設定し、ハンターを往復させます。">

<Concept title="巡回地点を表すTarget Point">

Target Pointは、AIの移動先として使う位置を表すActorです。

</Concept>

<Action title="巡回地点を配置する">

Place Actorsで`Target Point`を2つ配置します。どちらもNavigation Meshの緑色の範囲に置いてください。

| 巡回地点 | 配置する場所           |
| -------- | ---------------------- |
| 1つ目    | ステージ上側の左側     |
| 2つ目    | ステージ上側の右側     |

</Action>

<Action title="巡回地点の名前を変更する">

World Outlinerで、2つのTarget Pointを次の名前に変更します。

```text
PatrolPointA
PatrolPointB
```

</Action>

<Action title="巡回地点の変数を作る">

`BP_Hunter`に次の変数を作り、`Instance Editable`を有効にします。

| 変数名         | 型                              |
| -------------- | ------------------------------- |
| `PatrolPointA` | `Target Point Object Reference` |
| `PatrolPointB` | `Target Point Object Reference` |

</Action>

<Action title="追跡状態の変数を作る">

Boolean型の変数`IsChasingPlayer`を作ります。初期値は`false`です。

- `false`：巡回する
- `true`：プレイヤーを追いかける

</Action>

<Action title="変数を表示できるようにする">

`BP_Hunter`をCompileします。

</Action>

<Action title="巡回地点を割り当てる">

レベル上の`BP_Hunter`を選択し、Detailsで次のActorを割り当てます。

| 変数           | 割り当てるActor          |
| -------------- | ------------------------ |
| `PatrolPointA` | レベル上の`PatrolPointA` |
| `PatrolPointB` | レベル上の`PatrolPointB` |

</Action>

<Concept title="AI MoveToで目的地へ移動する">

`AI MoveTo`は、AIが指定したActorまで移動するノードです。

</Concept>

<Action title="巡回地点Aへ移動するイベントを作る">

`BP_Hunter`のEvent GraphにCustom Eventを追加して`MoveToPatrolPointA`に変更し、次のように接続します。

```text
MoveToPatrolPointA
    ↓
Branch
Condition: IsChasingPlayer

False
    ↓
AI MoveTo
Pawn: Self
Target Actor: PatrolPointA
Acceptance Radius: 50
```

`IsChasingPlayer`が`false`の間だけ、ハンターが`PatrolPointA`へ向かいます。

</Action>

<Action title="巡回地点Bへ移動するイベントを作る">

`MoveToPatrolPointA`と同じ構成で`MoveToPatrolPointB`を作り、`AI MoveTo`の`Target Actor`へ`PatrolPointB`を接続します。

</Action>

<Action title="2つの巡回イベントをつなぐ">

`PatrolPointA`へ到着したら`MoveToPatrolPointB`、`PatrolPointB`へ到着したら`MoveToPatrolPointA`を呼び出します。

```text
PatrolPointAへ到着
    ↓
MoveToPatrolPointB
    ↓
PatrolPointBへ到着
    ↓
MoveToPatrolPointA
```

</Action>

<Action title="ゲーム開始時に巡回を始める">

`Event BeginPlay`から`MoveToPatrolPointA`を呼び出します。

```text
Event BeginPlay
    ↓
MoveToPatrolPointA
```

</Action>

<Action title="巡回処理を保存する">

`BP_Hunter`をCompileしてSaveします。

</Action>

<Action title="巡回速度を調整する">

Playして巡回速度を確認し、必要なら`Character Movement`の`Max Walk Speed`を調整します。

</Action>

<Verify title="巡回を確認する">

- ハンターが`PatrolPointA`と`PatrolPointB`を往復する
- 壁がある場所では、壁を回り込んで移動する

</Verify>

<Recovery title="ハンターがまったく動かないとき">

- `BP_Hunter`と2つの巡回地点がNavigation Mesh上にあるか確認する
- レベル上の`BP_Hunter`に、2つの巡回地点が割り当てられているか確認する
- `Event BeginPlay`から`MoveToPatrolPointA`を呼び出しているか確認する
- `AI MoveTo`の`Pawn`に`Self`が接続されているか確認する

</Recovery>

<Recovery title="片方の巡回地点で止まるとき">

- 止まった側の`AI MoveTo`の`On Success`が、反対側のイベントにつながっているか確認する
- 次の巡回地点までNavigation Meshがつながっているか確認する
- `Target Actor`に正しい巡回地点の変数が接続されているか確認する
- 目的地まで移動できないと`On Success`が実行されず、次の巡回へ進まない

</Recovery>

<Checkpoint title="理解度確認2">

`PatrolPointA`がNavigation Meshの外にあると、ハンターの巡回はどうなりますか。

<details>
<summary>ヒント</summary>

`AI MoveTo`が目的地まで移動できなかった場合を考えます。

</details>

<details>
<summary>答え</summary>

ハンターは`PatrolPointA`へ到達できません。`AI MoveTo`が失敗して`On Success`が実行されないため、次の巡回イベントへ進まず、その場で止まります。

</details>

</Checkpoint>

</Section>

<Section title="警戒エリアに入ったら追跡を始める" goal="プレイヤーが警戒エリアに入ると、ハンターが追いかけてくるようにします。">

<Action title="警戒エリアのBlueprintを作る">

`Content/NightEscape/Blueprints`に、`Actor`を親にした`BP_AlertZone`を作ります。

</Action>

<Action title="警戒範囲を追加する">

`BP_AlertZone`に`Box Collision`を追加し、名前を`AlertCollision`に変更します。

</Action>

<Action title="警戒範囲を設定する">

`AlertCollision`を次のように設定し、図の警戒エリア全体を囲む大きさに広げます。

| 項目                      | 値                |
| ------------------------- | ----------------- |
| `Collision Presets`       | `OverlapOnlyPawn` |
| `Generate Overlap Events` | 有効              |

</Action>

<Action title="Hunter変数を作る">

`BP_AlertZone`に`BP_Hunter Object Reference`型の変数`Hunter`を作り、`Instance Editable`を有効にします。

</Action>

<Action title="Hunter変数を表示できるようにする">

`BP_AlertZone`をCompileします。

</Action>

<Action title="警戒エリアを配置する">

`BP_AlertZone`を図の警戒エリアへ配置し、`PlayerStart`は範囲外に残します。

</Action>

<Action title="ハンターを割り当てる">

レベル上の`BP_AlertZone`を選択し、Detailsの`Hunter`にレベル上の`BP_Hunter`を割り当てます。

</Action>

<Action title="ハンターに追跡処理を作る">

`BP_Hunter`にCustom Eventを追加して`StartChasingPlayer`に変更し、次のように接続します。

```text
StartChasingPlayer
    ↓
Set IsChasingPlayer
Value: true
    ↓
Set Light Color
Target: HunterLight
New Light Color: 赤
    ↓
AI MoveTo
Pawn: Self
Target Actor: Get Player Character
Acceptance Radius: 80
Stop on Overlap: true
```

`Target Actor`にプレイヤーを指定するため、移動中のプレイヤーを追跡します。

</Action>

<Action title="Overlapイベントを追加する">

`BP_AlertZone`のEvent Graphで、`AlertCollision`の`On Component Begin Overlap`を追加します。

</Action>

<Action title="プレイヤーか確認する">

`Other Actor`ピンから`Cast To BP_ThirdPersonCharacter`を追加します。

</Action>

<Action title="追跡を始める">

Cast成功側から、`Hunter`の`StartChasingPlayer`を呼び出します。

</Action>

<Action title="警戒判定を無効にする">

`StartChasingPlayer`のあとに`AlertCollision`の`Set Collision Enabled`を接続し、`New Type`を`No Collision`にします。追跡開始の重複を防ぐため、判定は1回で無効にします。

</Action>

完成した接続は次のようになります。

```text
AlertCollision
On Component Begin Overlap
    ↓
Cast To BP_ThirdPersonCharacter
Object: Other Actor
    ↓
Hunter
StartChasingPlayer
    ↓
Set Collision Enabled
Target: AlertCollision
New Type: No Collision
```

<Verify title="巡回から追跡に変わるか確認する">

- 警戒エリアの外では、通常色のハンターが巡回する
- 警戒エリアへ入ると、ライトが赤くなる
- ハンターが巡回をやめて追いかける

</Verify>

<Recovery title="警戒エリアに入っても追いかけてこないとき">

- `BP_AlertZone`の`Hunter`に、レベル上の`BP_Hunter`が割り当てられているか確認する
- `AlertCollision`の`Collision Presets`が`OverlapOnlyPawn`になっているか確認する
- `AlertCollision`の`Generate Overlap Events`が有効か確認する
- `Other Actor`がCastの`Object`につながっているか確認する
- Cast成功側から`StartChasingPlayer`を呼び出しているか確認する
- `AI MoveTo`の`Target Actor`に`Get Player Character`の`Return Value`が接続されているか確認する

</Recovery>

<Checkpoint title="理解度確認3">

追跡を始めるとき、最初に`IsChasingPlayer`を`true`にするのはなぜですか。

<details>
<summary>ヒント</summary>

巡回イベントの`Branch`が、どちらへ進むかを考えます。

</details>

<details>
<summary>答え</summary>

巡回イベントが再び呼び出されても、`Branch`のFalse側へ進まず、巡回用の`AI MoveTo`を実行しないようにするためです。

</details>

</Checkpoint>

</Section>

<Section title="警戒開始の演出を追加する" goal="プレイヤーが見つかると入口が閉じる演出を追加します。">

### 演習1

<Exercise>

プレイヤーが警戒エリアに入ると、通ってきた入口のシャッターが閉じるようにしてください。

次の状態になれば完成です。

- Play開始時はシャッターが開いている
- 警戒エリアに入るとハンターが追跡を始める
- プレイヤーの後ろでシャッターが閉じる
- 閉じるシャッターがプレイヤーに重ならない

<Hint>

まず、Cubeが開いた位置から閉じた位置へ動くBlueprintを作ります。動く床やエレベーターで使った、Timelineで位置を変える処理を利用できます。

</Hint>

<Hint>

シャッターをレベルへ配置したら、`BP_AlertZone`からそのシャッターを呼び出せるようにします。`BP_AlarmShutter Object Reference`型の変数を作り、`Instance Editable`を有効にすると、Detailsから配置したシャッターを割り当てられます。

</Hint>

<Answer>

`BP_AlarmShutter`を作り、入口をふさぐ大きさのCubeを追加します。Timelineでは、シャッターが開いている位置と閉じている位置を補間し、更新中にCubeの位置を変更します。Custom Eventの`CloseShutter`からTimelineを再生できるようにします。

`BP_AlertZone`には`BP_AlarmShutter Object Reference`型の変数`AlarmShutter`を作り、`Instance Editable`を有効にします。レベルへシャッターを配置したあと、`BP_AlertZone`のDetailsで`AlarmShutter`にそのシャッターを割り当てます。

Cast成功後の処理は、次の順にします。

```text
Cast成功
    ↓
Hunter
StartChasingPlayer
    ↓
AlarmShutter
CloseShutter
    ↓
Set Collision Enabled
Target: AlertCollision
New Type: No Collision
```

シャッターは警戒エリアの入口より手前に置き、プレイヤーが通過してから閉じるようにします。これにより、追跡開始と入口封鎖が同時に起こり、プレイヤーはステージの奥へ逃げる必要が生まれます。

Playして、シャッターが開いた状態から始まり、警戒エリアへ入ったときだけ1回閉じることを確認します。

</Answer>

</Exercise>

<Checkpoint title="完成チェック">

- 床がすべてNavigation Meshの緑色で表示されている
- Play開始時に、通常色のハンターが2つの地点を往復する
- 警戒エリアに入ると、ハンターのライトが赤くなる
- ハンターが巡回をやめ、プレイヤーを追いかける
- ハンターが壁を回り込んで移動する
- 警戒エリアに入ると、入口のシャッターが閉じる

</Checkpoint>

ここからは、時間に余裕がある場合に取り組みます。

### 演習-発展1

<Exercise>

`PatrolPointC`を追加し、ハンターが3つの地点をA→B→C→Aの順に巡回するようにしてください。

`PatrolPointC`は、AとBを往復するだけでは通らない場所へ配置します。警戒エリアに入ると巡回をやめ、プレイヤーを追いかければ完成です。

<Hint>

`PatrolPointA`を追加したときと同じ順序で考えます。レベルへTarget Pointを置き、`BP_Hunter`にそのTarget Pointを受け取る変数を作ります。

</Hint>

<Hint>

`MoveToPatrolPointA`の`Branch`と`AI MoveTo`の構成を確認してください。同じ構成でCへ移動するイベントを作ったあと、到着時の呼び出し順がA→B→C→Aになるようにつなぎ直します。

</Hint>

<Answer>

レベルへ3つ目のTarget Pointを配置して`PatrolPointC`に変更します。`BP_Hunter`にも`Target Point Object Reference`型の`PatrolPointC`変数を作り、`Instance Editable`を有効にしてCompileしたあと、レベル上の`PatrolPointC`を割り当てます。

`MoveToPatrolPointC`は、既存の巡回イベントと同じ構成にします。

```text
MoveToPatrolPointC
    ↓
Branch
Condition: IsChasingPlayer

False
    ↓
AI MoveTo
Pawn: Self
Target Actor: PatrolPointC
Acceptance Radius: 50
```

3つのイベントの`On Success`を次のようにつなぎます。

```text
MoveToPatrolPointA
    ↓
MoveToPatrolPointB
    ↓
MoveToPatrolPointC
    ↓
MoveToPatrolPointA
```

すべての巡回イベントに`IsChasingPlayer`のBranchを残すことで、警戒エリアへ入ったあとは巡回用の`AI MoveTo`が実行されません。PlayしてA→B→C→Aの順に巡回し、警戒後はプレイヤー追跡へ切り替わることを確認します。

</Answer>

</Exercise>

### 演習-発展2

<Exercise>

プレイヤーを追跡している間だけ、ハンターが巡回中より速く走るようにしてください。

警戒エリアに入る前後で速さの違いが分かり、速くなったあとも壁を回り込んで追跡できれば完成です。

<Hint>

ハンターの移動速度は、`BP_Hunter`の`Character Movement`にある`Max Walk Speed`で決まります。速度を変える処理は、巡回から追跡へ切り替わる場所に追加します。

</Hint>

<Hint>

`StartChasingPlayer`を開き、Componentsパネルの`Character Movement`をEvent Graphへドラッグします。そこから`Set Max Walk Speed`を追加し、追跡用の`AI MoveTo`より前に接続します。

</Hint>

<Answer>

`BP_Hunter`のComponentsパネルから`Character Movement`をEvent Graphへドラッグし、`Set Max Walk Speed`を追加します。`StartChasingPlayer`の`Set Light Color`のあと、追跡用の`AI MoveTo`より前へ接続します。

```text
Set Light Color
    ↓
Set Max Walk Speed
Target: Character Movement
New Max Walk Speed: 巡回中より大きい値
    ↓
AI MoveTo
Target Actor: Get Player Character
```

追跡開始時にだけ最大歩行速度を書き換えるため、警戒エリアへ入る前は巡回用の速度、入ったあとは追跡用の速度になります。Playして通常色で巡回している間と、赤色になって追跡している間の速さを比較します。速度を上げてもNavigation Meshに沿って壁を回り込めれば完成です。

</Answer>

</Exercise>
